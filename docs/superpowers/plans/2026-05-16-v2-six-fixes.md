# V2 Six Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the six owner-reported v2 fixes in two waves of parallel single-task dispatches.

**Architecture:** Reuse-first. Fix A consolidates status-change UX into the existing `StatusChangeTriggerDialog`. Fix B normalises calendar to a two-marker model on a single (effective) pickup date. Fix C does a real schema rename + label sweep. Fix D adds two derived money sums. Fix E pivots compose to an HTML-template wrapper. Fix F is a coordinated rip-and-replace of five drawer subcomponents.

**Tech Stack:** Spring Boot 3.4 / Java 21 / Flyway / Postgres 16 (backend); Next 16 / React / Tailwind / Radix / vitest (frontend); Playwright (e2e).

**Source of truth for the why/what:** `docs/superpowers/specs/2026-05-16-v2-six-fixes-design.md`. Each task below points at the relevant spec section and avoids restating the design.

**Dispatch contract:** Thin prompts referencing this plan file + task id. Each task is one Sonnet dispatch, one atomic commit, one dispatch-log entry under `docs/dispatch-log/v2-<id>-<UTC>.md`. Commit tag: `[milestone:v2-fixes][task:v2-<id>]`. Combined single-stage review except where marked **[TWO-STAGE]**.
