## Decisions

1. **NBSP normalisation in `formatPln`:** `NumberFormat.getCurrencyInstance(Locale.forLanguageTag("pl-PL"))` produces U+00A0 (non-breaking space) as the thousands separator AND between the number and "zł" (e.g. `"18 240,00 zł"`). The plan-specified test strings use regular spaces (U+0020). The controller normalises by calling `.replace(' ', ' ')` so the JSON output is copy-paste friendly and matches the plan's acceptance criteria exactly. This is documented in the commit message and this log.

2. **Package placement:** Plan stub paths used `backend/app/src/...` which matches reality (the module is `app`, not `admin-monolith` — the latter is the Maven artifactId alias used in `-pl` flags). Created `com.drshoes.app.dashboard.{api,dto}` following the existing `order.api` / `order.dto` convention.

3. **No migration needed:** All three repository queries are JPQL against existing columns (`status`, `deleted_at`, `received_at`, `total_price_cents`). No V014 migration required.

4. **Timezone semantics:** All window boundaries are computed in Europe/Warsaw via `ZonedDateTime.now(WARSAW)`. `todayIntakeCount` counts orders with `receivedAt ∈ [todayStart, tomorrowStart)` in Warsaw time, consistent with the plan. `monthRevenueCents` counts `receivedAt ∈ [monthStart, firstDayNextMonth)`.

5. **COALESCE in sumRevenueBetween:** The JPQL uses `COALESCE(SUM(...), 0)` so an empty window returns 0 rather than NULL, avoiding an NPE on the `long` return type.

6. **No `@Audited` annotation on the controller method:** The endpoint is GET (read-only). The existing `AuditLogAspect` audits it automatically via path-pattern matching (`/api/admin/**`), producing one audit row per request as observed in test logs.
