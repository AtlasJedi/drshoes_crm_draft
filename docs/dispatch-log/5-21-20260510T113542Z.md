## Decisions

### Backend flake — MessagesControllerIntegrationTest.emptyThreadReturnsEmptyList

Root cause: `AdminWebTestBase.seedUsers()` (@BeforeEach in base class) calls `clients.deleteAll()` at line 54. When this test class runs after another test class that also seeds a client + order, if the ordering of JUnit cleanup leaves an order still referencing a client from the previous class's test, the delete fails with FK violation (`order__client_id_fkey`). The error fires in `@BeforeEach` (not @Test body), which is why the test reports ERROR not FAILURE.

This flake was tracked since task 5-16 ("new flake in MessagesControllerIntegrationTest tracked for M5 closure hygiene" — session memory `project_session_2026_05_10_part5.md`). The dispatch spec for task 5-21 explicitly states: "If exactly that one test flakes intermittently and the rest are green, document it in the dispatch log and continue — that's a known issue."

Fix path for M6 hygiene: `AdminWebTestBase.seedUsers()` needs to delete orders before clients (add `orderRepository.deleteAll()` call or cascade), matching the pattern already applied in subclass `@AfterEach`. This is a test-infrastructure issue, not a production code defect.

During `pnpm build`, Next.js logs ERROR-level messages for `/admin/templates` and `/admin/triggers` because the static-generation worker hits the `cookies()` call.
