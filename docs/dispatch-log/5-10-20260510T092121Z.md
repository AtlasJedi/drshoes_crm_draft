## Decisions / deviations from plan

1. **sendReply not shipped by 5-3**: Plan stated `MessageRouter.sendReply` would be added by slice-A task 5-3. It was not. Rather than stub with `UnsupportedOperationException` (which would make happy-path ITs fail), `sendReply` was implemented here in `MessageRouter`. Signature differs slightly from plan — takes explicit `clientId` param because the controller pre-validates the thread and has the clientId already, avoiding a redundant DB round-trip in MessageRouter.

2. **Test base changed to AdminWebTestBase**: Plan's test used `SecurityMockMvcRequestPostProcessors.user(...)` with a synthetic UUID. This fails because `message.sent_by` and `audit_log.actor_id` are FKs to `user_`. Switched to `AdminWebTestBase` which seeds real user rows, satisfying FK constraints. The `owner()` helper is replaced by `loginAsOwner()` from the base class.

3. **CSRF required**: All POST calls in the IT needed `.with(csrf())`. The plan's test omitted this. Added.
