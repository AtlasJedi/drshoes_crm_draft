## Decisions

1. **Mutable mock string pattern** — `vi.mock` is hoisted and runs once per
   module. Per-test URL param variation is achieved by a module-level
   `let mockSearchParamsString = ""` read at render time inside the mock
   factory. This avoids `vi.doMock` which does not re-import an already-loaded
   module within the same test file.

2. **`SELECTABLE_STATUSES` hardcoded** — rather than deriving from `STATUS_ORDER`
   with a filter, the selectable set is explicit to make the exclusion of
   `WSTEPNIE_PRZYJETE` obvious and easy to audit. Defensive label computation
   still covers it via `STATUS_LABELS_PL` fallback.

3. **`onSelect` called with empty array on Wyczyść** — caller (`OrdersFilters`)
   maps empty array → `push({ status: undefined })` which deletes the param.
   Keeps the component stateless/pure.

4. **`hasAnyFilter` checks 8 keys** — covers all filter dimensions named in the
   plan spec: status, type, craftsmanId, q, tag, plannedPickupAt*, sort.

5. **Toggle-off goes to `/admin/orders` bare** — no query string at all, matching
   the spec ("URL becomes /admin/orders (no query)").

6. **TypeScript `vi.fn` typing** — `vi.fn<(statuses: OrderStatus[]) => void>()`
   required to satisfy `Props.onSelect: (statuses: OrderStatus[]) => void`.
   Using `ReturnType<typeof vi.fn>` was too broad and caused tsc errors.
