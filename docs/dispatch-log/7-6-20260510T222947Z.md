## Decisions / Surprises

- Plan's GREEN implementation included `role="textbox"` explicitly on the `<input>` element. Native `<input type="text">` carries the implicit ARIA role `textbox`, so the explicit attribute is redundant and would trigger an `jsx-a11y/no-redundant-roles` lint warning. Removed it — tests use `screen.getByRole("textbox")` which still finds the element via implicit role. Zero deviation in behavior.
- No structured logging added — this is a pure UI CC with no side effects beyond router push. `lib/log.ts` not imported (consistent with other CC components like `OrdersWeekChart`, `KpiTilesRow`).
- `clients/_components/` directory and `__tests__/` subdirectory did not exist yet — created as part of this task (the plan implies they should be created).
- `initialQ` prop initialises `useState`, so the field reflects URL state on navigation (e.g. from SSR-rendered page that reads `searchParams.q`).
