## Deviations from prompt

1. **`music-storage.ts` debounce semantics** (`apps/web/lib/music-storage.ts:44`): Prompt said "max 1× per second". Initial implementation used a throttle (drop-if-pending) which caused the test to fail — the initial mount write would block the post-pick write. Corrected to a true debounce (clear+reschedule), which is correct last-write-wins semantics and still enforces ≤1 write/second.

2. **`AdminSidebar` now inside `.adm-side` div** (`apps/web/app/(admin)/admin/layout.tsx`): The prompt said to restructure the CSS grid but didn't specify wrapper element names. Added `.adm-side` and `.adm-main` wrapper divs as children of `AdminMusicShell` to map to grid areas — this is the minimal structural change needed without touching `AdminSidebar` itself.

3. **`PersistentMiniPlayer` included in test wrapper**: The prompt said tests wrap `MusicClient` in `<MusicProvider>`. Since the YouTube iframe now lives in `PersistentMiniPlayer` (not in `MusicClient`), the `data-testid="yt-stub"` assertions required `PersistentMiniPlayer` in the tree. Added it to `renderWithProvider()` — behaviour under test is identical, only the component boundary moved.
