## Decisions / deviations from plan

1. **V003 is a no-op comment migration.** The plan spec showed `V003__clients.sql` creating the `client` table from scratch. However, `V001__init.sql` is a comprehensive baseline that already fully defines `client` (including indexes, trigger, and FK references from `order_`, `message_thread`, `scheduled_message`, etc.). Attempting to re-create the table caused `ERROR: relation "client" already exists` on the Flyway migration run. V003 was converted to a comment-only marker. This is the correct approach — V001 is the single baseline migration and all subsequent migrations are incremental changes.

2. **Client.java mapped to actual V001 schema.** The plan's `Client.java` spec had a simplified schema (TEXT columns, last_name NOT NULL, no preferred_channel/rodo_consent_at). The entity was updated to match the real V001 schema: `first_name VARCHAR(80) NOT NULL`, `last_name VARCHAR(80)` (nullable), `email CITEXT`, `preferred_channel VARCHAR(16) NOT NULL DEFAULT 'EMAIL'`, `rodo_consent_at TIMESTAMPTZ`, plus the `updatable=false` annotation on `createdAt`.

3. **Test helper fixed for `client_contact_present` constraint.** The plan's `newClient(f, l)` helper created clients with no phone or email, violating the DB-level CHECK constraint `(phone IS NOT NULL OR email IS NOT NULL)`. Added a default phone `+48 000 000 000` to the helper.

4. **Test base class: `AbstractIntegrationTest` not `PostgresIntegrationTestBase`.** The plan referenced `com.drshoes.app.test.PostgresIntegrationTestBase` which does not exist. Used the actual base class `com.drshoes.app.AbstractIntegrationTest`. Removed the redundant `@SpringBootTest` annotation from the test class (the base already carries it).

5. **`Client.java` is 78 LOC** — within the entity exception documented in the plan ("Java entities are an explicit exception in the dispatch-protocol granular-code rule").

6. **`searchTopN` uses LIKE** per plan spec (gin index from V001 is for production; LIKE is for test simplicity). Documented per plan instruction.
