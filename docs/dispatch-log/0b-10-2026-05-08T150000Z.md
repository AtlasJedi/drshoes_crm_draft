## Decisions / deviations from plan

- **`cookies()` is async in Next.js 16**: The plan's `session.ts` used `const c = cookies()` (sync). Next.js 16 made `cookies()` return a Promise. Changed to `const c = await cookies()` — required to avoid TypeScript error and runtime breakage.
- **`any` in log.ts**: One `(globalThis as any)` cast at the `__LOG_LEVEL__` check is unavoidable — `globalThis` has no typed `__LOG_LEVEL__` property. Narrowed with a cast at the boundary and documented with eslint-disable comment (though no eslint config exists).
- **Plan imported `headers` from next/headers in session.ts** but never used it. Omitted to keep file clean.
- **lib/log.ts addition**: Not in plan body, but required by dispatch protocol. Added as the named-logger module all substantive TS modules must use.

## Logging / metadata wired
- Module `lib/log.ts` exports `createLogger(name)` and default `log = createLogger("web")`
- Module `lib/api.ts` uses `log = createLogger("api")`: logs `warn` on non-2xx with `op`, `path`, `status`, `requestId`; logs `info` on 401 redirect with same fields
- Module `lib/auth/session.ts` uses `log = createLogger("auth:session")`: logs `info` on no-session / unauthorized, `warn` on unexpected error with `status`
