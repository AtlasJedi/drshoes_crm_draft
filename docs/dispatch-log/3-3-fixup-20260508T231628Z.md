# Task 3-3 fixup — AdminPrincipalLoginTest FK teardown

## Context

Stage-2 code-quality reviewer flagged one IMPORTANT issue on commit `24e533e`
(M3 task 3-3): `AdminPrincipalLoginTest` did not clean up `audit_log` rows
in `@BeforeEach`/`@AfterEach`. After 3-3 wired `audit_log.actor_id` to a real
user UUID, calling `users.deleteAll()` would violate the
`audit_log_actor_id_fkey` FK whenever the test had run `authService.login()`
(which produces an audited HTTP-equivalent row through `AuditLogAspect`).

Three other tests touched in 3-3 (`AdminWebTestBase`, `AuthControllerIntegrationTest`,
`RbacIntegrationTest`) had this teardown order patched. `AdminPrincipalLoginTest`
was missed. This fixup brings it in line.

## Files changed

- `backend/app/src/test/java/com/drshoes/app/auth/principal/AdminPrincipalLoginTest.java` (modified)
  - Added `@Autowired AuditLogRepository auditLogs`.
  - `seed()` now calls `auditLogs.deleteAll()` before `users.deleteAll()`.
  - `cleanup()` mirrors the same order.
  - Added a one-line comment explaining the FK ordering rule.
