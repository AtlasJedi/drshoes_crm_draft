## Decisions

1. **Design-block override (BLOCK gate at plan line 7227-7229):** Owner explicitly overrode the `handoff/design/m6-kanban-states.html` gate — same override as 6-12 and 6-16. Drag ghost styling uses token-derived rules from the dispatch prompt: `opacity-90 rotate-1 shadow-lg` on the `DragOverlay` wrapper; source card `opacity-40` is handled by `KanbanCard.tsx`'s existing `isDragging ? 0.4 : 1` opacity via `useSortable`. Drop-target column highlight (`ring-2 ring-[var(--pink)] ring-offset-2`) is NOT applied in this stage — the plan's Step 6 said to fill in from the design export and `KanbanColumn.tsx` already uses `bg-black/10` on `isOver`. This is a Stage 2 focus item.

2. **`busy` state removed from `OrderDrawerStatusChanger`:** The old `StatusChangeConfirm` used `busy` for a `disabled` button state. `StatusChangeTriggerDialog` has no `busy` prop by design (the drawer's flow fires `handleConfirm` asynchronously after dialog dismissal). Removing `busy` eliminated a lint warning and simplified the component.

3. **`onConfirm` callback in drawer ignores `sendTriggers`:** Per plan step 4: "The `_sendTriggers` value is intentionally ignored for the drawer path in M6 (wiring it fully is M7 scope)." Changed from `(_sendTriggers) => ...` to `() => ...` to satisfy `--max-warnings=0` lint.

4. **Functional state updater in `onDragEnd`:** Used `setColumns((prev) => { ... })` with side-effectful `setSnapshot` and `setPendingMove` calls inside the updater body. This is unconventional but React 18 batches all synchronous state updates within the same event handler, so all three state updates (`columns`, `snapshot`, `pendingMove`) land in the same batch. Tests confirm correct behavior.

5. **`KanbanBoard` props made optional:** `onDragEnd`, `pendingMove`, `onConfirm`, `onCancel` are all optional with defaults so existing `KanbanBoard` call sites (e.g. the kanban page from 6-18) don't break without a parent providing hook results. The page wiring (passing `useKanbanDnd` results into `KanbanBoard`) is not part of 6-19 scope — it belongs to the kanban page component from 6-18, which will need updating in a follow-up.

6. **`KanbanBoard.test.tsx` mock update:** The existing mock didn't include `DragOverlay` (6-18 didn't use it). Adding it to the mock was an out-of-plan fix required by the regression — documented here per template §9.
