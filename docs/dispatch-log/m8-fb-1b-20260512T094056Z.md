## Decisions

1. **Interface approach chosen over reflection** — `HasAuditNote` is an explicit contract.
   `ChangeStatusRequest.auditNote()` delegates to `note()`. Zero reflection overhead on
   audit writes for DTOs that don't implement the interface.

2. **HTTP audit path for note** — `changeStatus` in `OrderService` is NOT `@Audited`;
   its audit row comes from the HTTP aspect. So note extraction happens in
   `AuditLogAspect.audit(pjp)` which has access to `pjp.getArgs()` including the
   `ChangeStatusRequest`.

3. **No index on `audit_log.note`** — as directed by owner (locked spec).

4. **Empty-string → omit from JSON** — frontend trims and omits empty note so backend
   receives `null` rather than `""`, keeping the DB column semantically null.

5. **`_sendTriggers` param unused in StatusChanger** — the changer always sends triggers
   based on user choice, but the note is forwarded regardless of trigger choice. The
   `sendTriggers` boolean was already handled by the pre-existing flow; only note
   needed threading.

6. **Service-layer test vs controller test** — `OrderServiceIntegrationTest` tests DTO
   contract (`auditNote()` returns expected value). Actual DB persistence of `audit_log.note`
   requires HTTP context (aspect fires at controller boundary), so is tested in
   `OrderControllerIntegrationTest` via JdbcTemplate.

7. **Frontend test count increase** — 15 new tests (vs 188 baseline = +8%). Pre-existing
   2-file e2e/Playwright failure in Vitest is a carry-forward hygiene item (Vitest picks up
   files in `e2e/` directory that use Playwright test API). Not introduced by this task.
