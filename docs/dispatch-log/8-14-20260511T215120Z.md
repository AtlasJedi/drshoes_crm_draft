## Decisions

1. **`resourceFromAttributes` not available** — `@opentelemetry/resources@1.28.0` does not export `resourceFromAttributes`. Used `new Resource({})` instead (the stable API). Documented as out-of-plan errata.

2. **`ATTR_DEPLOYMENT_ENVIRONMENT_NAME` not available** — `@opentelemetry/semantic-conventions@1.28.0` exports `SEMRESATTRS_DEPLOYMENT_ENVIRONMENT` (the older semresattrs prefix form) but not `ATTR_DEPLOYMENT_ENVIRONMENT_NAME`. Used `SEMRESATTRS_DEPLOYMENT_ENVIRONMENT` rather than a bare string literal to keep type safety. Confirmed via `node -e "require('./apps/web/node_modules/@opentelemetry/semantic-conventions')"`.

3. **Auto-instrumentation selective keys skipped** — Plan suggested passing `{ "@opentelemetry/instrumentation-http": { enabled: true }, ... }` but per errata this can throw on key mismatch. Verified `getNodeAutoInstrumentations({})` works (38 instrumentations). Passed empty config to accept all defaults — includes HTTP and fetch instrumentations with no runtime risk.

4. **`@vitest-environment node` annotation** — Global vitest config sets `environment: "jsdom"`. OTel packages are Node-only (import `perf_hooks`, `async_hooks`, etc.) and fail in jsdom. Added `// @vitest-environment node` doc comment at top of `instrumentation.test.ts` to override per-file. Both tests pass.

5. **No `next.config.mjs` change** — Confirmed: Next.js 15+ (including 16) enables the instrumentation hook by default. No `experimental.instrumentationHook` flag needed.

6. **LOC** — `instrumentation.ts` is 44 LOC, well under the 80 LOC budget.
