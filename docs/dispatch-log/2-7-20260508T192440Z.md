## Microlib adaptation decisions (ADAPT-POINTS)

**1. OutboundMessage — no builder. Plan used `OutboundMessage.builder()`.**
Actual class is a plain Java record: `OutboundMessage(Channel channel, String recipient, String subject, String body, List<Attachment> attachments, String idempotencyKey)`.
Adapted: `new OutboundMessage(ch, recipient, renderedSubject, renderedBody, null, null)`.
`null` attachments is accepted by the compact constructor (coerced to `List.of()`).
`null` idempotencyKey is accepted.

**2. `@Audited` attribute name. Plan used `@Audited(parentEntityId = "#orderId")`.**
Actual annotation has `parent()` not `parentEntityId()` (see `Audited.java` line 29).
Adapted: `@Audited(parent = "#orderId")`.
SpEL binding uses `-parameters` flag (confirmed in tasks.json 1-8 key_decisions); `#orderId` resolves by parameter name.

**3. Entity class names. Plan imported `OrderEntity` and `ClientEntity`.**
Actual classes are `Order` and `Client` (no `Entity` suffix — consistent with M1 codebase).

**4. Order.getItems() does not exist.**
`Order` entity has no items collection (aggregate boundary: items fetched separately via `OrderItemRepository`).
Adapted: `orderItems.findAllByOrderIdOrderByPosition(orderId)` — injected `OrderItemRepository` as additional dependency.

**5. `plannedPickupAt` returns `Instant` not `OffsetDateTime`.**
`TemplateContext.dataOdbioru` expects `OffsetDateTime`. Converted with `instant.atOffset(ZoneOffset.UTC)`. If null, passes null through (renderer emits "—").

**6. `polishKindLabel` — takes `OrderItemKind` enum not String.**
`orderItem.getKind()` returns `OrderItemKind` enum. Used `switch (kind)` over enum values with exhaustive case (no default needed — compiler enforces).

**7. Provider ID prefix. Plan test asserted `startsWith("log-")`.**
`LoggingEmailGateway` actually returns `"logging-" + UUID.randomUUID()`. Test adapted to `startsWith("logging-")`.

**8. MessageEntity constructor was `protected`.**
Service is in package `messaging.service`, entity in `messaging.domain` — different packages, `protected` was inaccessible.
Changed to `public MessageEntity() {}`. Side effect: also added `@JdbcTypeCode(SqlTypes.JSON)` on the `attachments` field — without it Hibernate tried to bind the default `"[]"` as `character varying` against a `jsonb` column and every insert failed with a PSQL type error. Both changes to `MessageEntity.java` are non-behavioral.

## Logging contract
Every `send(...)` call emits exactly one INFO line:
```
op=message.send outcome=SENT|FAILED orderId={} messageId={} channel={} triggerId={}
```
On gateway exception a WARN is also emitted before the INFO (outcome=FAILED), so the INFO is always the final word per call. Confirmed in test run logs.

`@Audited(parent="#orderId")` fires after successful return from `sendManual` / `sendForTrigger`. Confirmed in test logs: `op=auditAnnotated target=MessageRouter#sendManual parentEntityId=<orderId>`.

## Concerns for reviewer

1. **`MessageEntity` constructor visibility** — changed `protected` → `public`. JPA only requires a no-arg constructor; visibility is a style concern. Standard practice in Spring Boot is `public` or `protected`. No functional risk, but reviewer should confirm this is acceptable given the project convention (all other entities use `protected`). Alternative: move `MessageRouter` to the `messaging.domain` package — reviewer may prefer that.

2. **`@JdbcTypeCode(SqlTypes.JSON)` on `attachments`** — this was missing from the entity as shipped in task 2-3. Without it all message inserts fail. Added as part of this task. This is a bug fix to a prior task's entity; reviewer should note it's a companion fix, not an out-of-scope change.

3. **Thread `lastMessageAt` bump** — `thread` entity comes from `MessageThreadService.findOrCreateForClient` which runs in its own `@Transactional`. After that call returns, the thread is a detached reference. The router saves it via `threads.save(thread)` in the outer transaction — this should work (JPA merge semantics) but reviewer should verify no stale-state or second-transaction issue in the multi-actor scenario.

4. **Null recipient on EMAIL channel** — if `client.email` is null, `OutboundMessage` constructor will throw `IllegalArgumentException("recipient must not be blank")` which is caught by the `catch (RuntimeException e)` block and stored as FAILED. This is reasonable behavior (spec item 7 from the review checklist) but not explicitly logged as a WARN before the general FAILED log. Reviewer may want a dedicated pre-check WARN.

5. **`sendForTrigger` audit path** — audit row path will be `MessageRouter#sendForTrigger`, not `MessageRouter#send`. The plan's ERRATA §5 says the curator should match `"MessageRouter#send"`. The private `send()` method is not annotated — `@Audited` is on the two public methods. Reviewer should confirm whether the curator (task 2-10) should match `MessageRouter#sendManual` and `MessageRouter#sendForTrigger` separately, or if `@Audited` should be moved to the private method (requires making it package-private or using a self-invocation proxy trick). This is an architectural decision for the reviewer.
