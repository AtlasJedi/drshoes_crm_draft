## Decisions / deviations from plan

1. **Added `@AfterEach cleanup()` to `V012MigrationIntegrationTest`** — the plan template did not include cleanup. The shared Testcontainers DB (withReuse=true, no transactional rollback in RANDOM_PORT context) caused orphan `message_thread` rows to remain after V012 tests. These rows blocked `MessageRetryControllerIntegrationTest.@BeforeEach seedUsers()` which calls `clients.deleteAll()` (FK violation: message_thread still references client). Added targeted DELETE statements keyed on distinguishable email patterns and body values. This is a deviation from the plan's exact code but is required for suite hygiene.

2. **Confirmed `ddl-auto: validate` is safe** — Hibernate validate mode does not enforce JPA `nullable=false` annotations against the DB at startup; it only validates that mapped columns exist with compatible types. The JPA entities (`MessageEntity`, `MessageThreadEntity`) still declare `nullable=false` on `client_id` — this will be patched in task 5-2 per plan. No Hibernate startup failure occurred.

3. **V012 confirmed as correct next version** — listed migration directory before writing; V011 was last.
