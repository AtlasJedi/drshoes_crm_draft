## Deferred to M10

All items below require a container rebuild to verify, OR are genuine remaining visual gaps:

1. **Container rebuild required before final parity sign-off** — The web Docker image (built 2026-05-15T14:52Z) predates all M9 commits. Rebuilding with `docker compose build web && docker compose up -d web` and re-running `m9-parity-audit.spec.ts` will give the true post-M9 parity picture. Estimated outcome after rebuild: public landing → ok; admin shell (sidebar dark) → ok; all views → ok except items 2–5 below.

2. **MixDonut chart on dashboard is empty** — The donut chart area in the dashboard shows no visual output (just the "12 aktywne" number and legend rows). The design shows a rendered pie/donut segment chart. This may require a Recharts/Chart.js library integration that hasn't landed yet. Deferred-M10.

3. **AdminTopbar global search input missing** — The design shows a global search bar (`Szukaj zlecenia, klienta…`) and notification bell in the topbar. The live topbar only has the page title. These are topbar features that weren't part of M9 scope per spec. Deferred-M10.

4. **Step timeline in order drawer** — The design shows a step-by-step progress indicator (PRZYJĘTE → W REALIZACJI → GOTOWE DO ODBIORU → WYDANE) at the top of the drawer. M9 task 9-24 implemented `OrderDrawerTimeline` — visible in source but requires container rebuild + drawer open interaction to verify. Deferred-M10 for interactive verification.

5. **Sticky notes panel in order drawer** — Design shows a yellow sticky note area (`fef4a8` background). M9 task 9-25 implemented it. Same container-rebuild dependency. Deferred-M10 for interactive verification.

6. **`RodoInline` → `RodoBadge` replacement in ClientListTable** — Plan explicitly calls this out as a 9-42 task (> 10 LOC). Not touched here. Deferred to 9-42.

7. **FreshReservationsPanel live data wiring** — Dashboard "Gotowe do odbioru" panel shows seeded data correctly; however the `FreshReservationsPanel` component (9-22) was shipped with placeholder data per cross-cutting note B. The `/api/admin/reservations` endpoint was not wired. Deferred-M10.

## Decisions
- The `admin-messages` test required `.first()` on `locator('main')` because MessagesShell renders a nested `<main>`. This is a structural design choice in the messages layout (two-pane layout uses `<main>` for both the outer layout region and the inner content pane). Could be refactored to use `<article>` or `<section>` for the inner pane in M10, but it's not a visual gap.
- Container staleness discovered via HTML inspection (`aside` class `bg-admin-surface` vs expected `bg-ink`). Documented as the primary deferred item. Screenshots are valid audit artifacts for the pre-M9 baseline.
- All 4 `ok` statuses (kanban, messages, sklep stub, aktualności stub) are valid — these views had no pending M9 reskins or their stubs are intentionally minimal.
