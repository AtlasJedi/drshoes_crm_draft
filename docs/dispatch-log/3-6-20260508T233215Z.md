## Decisions / deviations from plan

1. **`@Audited(parent=...)` not `parentEntityId=...`**: The plan body used `parentEntityId` as the annotation attribute name, but the actual annotation uses `parent()`. All `@Audited` calls use `parent=` as in the existing codebase.

2. **`#result` SpEL binding**: Plan's `@Audited(parentEntityId="#result")` on `delete` (returns UUID orderId) and `@Audited(parentEntityId="#result.orderId")` on `relabel` (returns Photo) require the return value to be bound in SpEL context. `AuditedParentResolver` only bound method args. Extended with a 4-arg overload that binds the return value as `#result`. `AuditLogAspect.auditAnnotated` updated to pass `out` to the resolver. The existing 3-arg `resolve(method, args, expr)` delegates to the 4-arg version — backwards compatible, existing test `spelEvaluatesAgainstArgs` still passes.

3. **`Photo` implements `Persistable<UUID>`**: `PhotoService.upload()` pre-generates `photoId` UUID before calling `photos.save()` so the S3 key can be built as `orders/{orderId}/{photoId}-{filename}`. With `@GeneratedValue`, Spring Data calls `merge()` on a non-null id and Hibernate throws `StaleObjectStateException`. Added `Persistable<UUID>` with `@Transient boolean isNew = true` cleared by `@PostPersist`/`@PostLoad`. `PhotoRepositoryTest` updated to call `p.setId(UUID.randomUUID())` in its helper.

4. **`PhotoService` is ~200 LOC**: Plan says 120 LOC cap with acceptable overrun for service classes (similar to M1 OrderController). All private helpers are small validators. No clean cohesion-based split available.

5. **`delete` semantics**: DB row deleted first (`photos.delete` + `photos.flush()`), then `BlobStorage.delete`. Storage failure logged as `orphan_failed` WARN and not propagated — matches plan's "best-effort delete on S3" policy.

6. **`TimelineEventCurator` uses `labels` map from parent `curateInternal`**: The plan snippet showed per-photo fields (`label`, `originalFilename`) in the curator labels map. However, the audit row has no payload JSON (the `AuditLog` entity has no payload capture in M1/M2). The curator passes the same `labels = Map.of("actorFullName", actorFullName, "orderId", orderIdLabel)` as existing item events — this is correct and consistent. The `KIND_LABELS_PL` mapping in task 3-12 provides the human-readable kind label; per-row params are best-effort and deferred.

7. **`PhotoServiceTest` seeds its own user**: Plan's test used `users.findAll().iterator().next()` to find the seeded owner. In the full test suite, `AdminWebTestBase` tests run `users.deleteAll()` in `@AfterEach` leaving no users. Fixed: `@BeforeEach` seeds a unique user per test run; `@AfterEach` deletes it by id.
