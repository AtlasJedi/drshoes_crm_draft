## Lift wiring decision

`MessagesShell.tsx` previously had `onLoaded={() => { /* sidebar refresh ... */ }}` as a no-op. Changed to `onLoaded={setLoadedThread}` where `loadedThread: MessageThreadDto | null` is held in MessagesShell state. `SelectedThread`'s `onLoaded` signature is `(t: MessageThreadDto) => void` — `setLoadedThread` matches exactly (React setState setter is `(value: T) => void`). No wrapper needed.

The right rail is mounted as a sibling after `<main>` inside the `flex flex-1 overflow-hidden` row, guarded by `sel.selectedId && loadedThread && loadedThread.clientId`. Unmatched threads (clientId null) never reach the panel mount.

## TypeScript fix during implementation

Plan draft used `client.firstName[0]` (string index) which TS strict mode rejects as `string | undefined`. Fixed to `client.firstName.charAt(0)` and `client.lastName.charAt(0)` — semantically identical, type-safe.
