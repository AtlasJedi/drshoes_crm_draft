## Allow-list (explicit, for audit)

The following request matchers are `permitAll()` (require no authentication):
1. `/actuator/health` — Spring Boot health probe
2. `/actuator/info` — Spring Boot info endpoint
3. `/api/public/**` — public-facing API routes (Milestone 5+)
4. `/api/webhooks/**` — inbound external webhook callbacks
5. `/api/admin/auth/login` — login endpoint (must be unauthenticated)
6. `/api/admin/auth/csrf` — CSRF token fetch endpoint (must be unauthenticated)

All other `/api/admin/**` routes require `authenticated()`.
Any other request (fallback `anyRequest()`) is `permitAll()` — covers frontend static assets, Next.js routes proxied through the API gateway, etc.

## CSRF configuration
- Mode: double-submit-cookie (`CookieCsrfTokenRepository.withHttpOnlyFalse()`)
- Cookie name: `XSRF-TOKEN` (httpOnly=false so frontend JS can read and re-submit in `X-XSRF-TOKEN` header)
- Request attribute name: `_csrf`
- CSRF ignored for: `/api/public/**`, `/api/webhooks/**`, `/actuator/**` (no state-changing ops on these routes)

## Logging / metadata wired
- `SecurityConfig` logs at INFO on startup: `op=securityFilterChainConfigured publicRoutes=6 csrfMode=double-submit-cookie csrfCookieName=XSRF-TOKEN`
- Uses SLF4J `LoggerFactory.getLogger(SecurityConfig.class)` — no string concat, key=value fields.
