## Decisions / deviations from plan

- **Polish labels assembled client-side.** The plan states "label assembly happens server-side; client just renders." However, inspecting `TimelineEventCurator.java`, the `labels` map contains raw metadata fields (`path`, `orderId`, `actorFullName`) — no composed Polish text. The curator did not implement a Polish label string; it only curated event kinds. Rendering readable Polish text therefore required a `KIND_LABELS_PL` map client-side. This is correct; the plan description was aspirational, not a DTO contract. Decision: thin `KIND_LABELS_PL` map in the component, one line per kind.

- **Emoji kind icons instead of Lucide.** `lucide-react` is not in `apps/web/package.json`. Used a `KIND_ICONS` emoji map (`🆕`, `🔄`, `➕`, etc.) as the plan explicitly offered "mini emoji" as a valid alternative. No new dependency added.

- **`refreshKey` lifted via `handleOrderUpdated` wrapper.** All three child callbacks (`OrderDrawerCoreFields.onOrderUpdate`, `OrderDrawerStatusChanger.onOrderUpdated`, `OrderDrawerItems.onOrderUpdated`) now go through `handleOrderUpdated` in `OrderDrawer.tsx`, which calls `setOrder` + `setRefreshKey(k => k+1)`. This matches the dispatch prompt's preferred shape ("wrap `onOrderUpdated` to also `setRefreshKey`").

- **106 LOC in `OrderDrawerTimeline.tsx`** — above the 80 LOC soft cap. The file contains: KIND_ICONS map (9 lines), KIND_LABELS_PL map (9 lines), `formatTs` helper (3 lines), the `load` callback, `useEffect`, and JSX with 3 conditional branches (loading / error / list). No meaningful split is possible without producing a trivial helper file for <10 lines. Acceptable overage per plan guidance ("If this comes in at 85, fine — don't fragment for fragmentation's sake").
