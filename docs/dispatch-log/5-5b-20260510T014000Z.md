## Problem

`AuditedParentResolver` uses `SimpleEvaluationContext.forReadOnlyDataBinding()` which blocks
method calls. Java record accessors (`threadId()`, `orderId()`) ARE method calls. Therefore
`@Audited(parent="#result.threadId()")` silently returned null → `parent_entity_id` was never
written for inbound message and retry audit rows, breaking timeline linkage.

Affected annotations:
- `InboundMessageService.recordEmailInbound` — `#result.threadId()`
- `InboundMessageService.recordSmsInbound` — `#result.threadId()`
- `MessageRetryService.retry` — `#result.orderId()`

## Option chosen: A (property access, no parens)

SpEL property access syntax (`#result.threadId` without `()`) delegates to
`SimpleEvaluationContext`'s `DataBindingPropertyAccessor`, which resolves JavaBean properties.
Spring Framework 6.x maps Java record component names to bean properties via the record's
accessor methods — so `threadId` resolves `threadId()` at the PropertyAccessor level without
needing method-call privileges.

PoC verified at the JVM level before applying the fix:
```
Option A (no parens): <uuid> | matches=true
Option parens FAILED: EL1004E: Method call: Method threadId() cannot be found on type ...
```
