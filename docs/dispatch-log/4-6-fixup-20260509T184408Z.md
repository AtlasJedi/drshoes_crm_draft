# Task 4-6 fixup — stage-2 review findings applied

## Findings (from stage-2 review at ff267e7)

1. **NEEDS_FIX — `SmsApiAutoConfiguration` ordering.** Add `before = SmsGatewayAutoConfiguration.class`. Without this, when `messaging.sms.provider=smsapi` is set the legacy `drshoes.sms.provider` is unset, so `LoggingSmsGateway`'s `@ConditionalOnProperty(matchIfMissing=true)` triggers; if `SmsGatewayAutoConfiguration` is processed first its `@ConditionalOnMissingBean(SmsGateway.class)` guard passes (SmsApi gateway not yet registered) and both beans land → `NoUniqueBeanDefinitionException` at startup. Identical pattern to 4-4's fixup #2.
2. **FLAG — Redundant method-level `@ConditionalOnProperty`.** Class-level annotation already gates the whole config; method-level duplicate on `smsApiSmsGateway()` is noise. Removed for consistency with the `smsApiRestClient` bean (which has no such duplicate).

## FLAG not actioned in this fixup

3. **Missing double-fault test case** (both retry attempts fail → `NETWORK` receipt). Real gap — the retry-then-give-up branch in `executeWithRetry` lines 75-81 is untested. Marked as M4 carry-forward debt rather than blocking 4-6 closure. Mirror of 4-4's PostmarkEmailGatewayIT 6th case. Track for a small test-only follow-up if any later task depends on retry-failure semantics; otherwise punt to M5.
