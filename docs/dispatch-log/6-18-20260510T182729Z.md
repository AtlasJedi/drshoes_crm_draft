## Decisions

1. **`@dnd-kit/utilities` installed separately** — not pulled transitively in jsdom env; `CSS.Transform.toString` caused a TS2307 error without it.

2. **`as Route` cast in `router.push`** — Next.js 16 typed router requires `RouteImpl<T>` for string URLs. Pattern matches `CalendarCell.tsx` (existing precedent). Import `type { Route } from "next"`.

3. **`KanbanBoard.test.tsx` mock patched** — plan-provided mock for `@dnd-kit/core` omitted `useDroppable`; `KanbanColumn` (rendered inside `KanbanBoard`) calls it, so added `useDroppable: () => ({ setNodeRef: vi.fn(), isOver: false })` to the board-level mock.

4. **Test assertion adjusted for `/wydane/i`** — the test fixture uses status names as card codes/client names (`DR-WYDANE`, `Klient WYDANE`), causing `getByText(/wydane/i)` to find multiple elements. Changed to `getAllByText(/wydane/i).length >= 1` for that column; other columns used exact label strings (`"Przyjęte"`, `"W realizacji"`, `"Gotowe do odbioru"`) which don't collide with card data.

5. **Lockfile at repo root** — `pnpm-lock.yaml` lives at `/pnpm-lock.yaml`, not `apps/web/pnpm-lock.yaml`. Staged from root.
