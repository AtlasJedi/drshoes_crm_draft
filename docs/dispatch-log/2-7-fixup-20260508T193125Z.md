# Task 2-7 — MessageRouter quality review + fixup

## Per-concern verdicts

### A — `MessageEntity` constructor visibility (`protected` → `public`)
**FIXED.** All other messaging entities (`TriggerEntity`, `TriggerFireEntity`,
`MessageTemplateEntity`) use `protected` no-arg constructors. `Order` and `Client`
have compiler-generated `public` defaults (no explicit constructor), so they are
not directly comparable. Changed `MessageEntity()` back to `protected` and added
`MessageEntity.newMessage()` static factory. `MessageRouter` now calls
`MessageEntity.newMessage()` instead of `new MessageEntity()`. JPA continues to
work: Hibernate uses reflection for proxy creation regardless of access modifier.

### B — `@JdbcTypeCode(SqlTypes.JSON)` missing from `TriggerEntity.eventParams` and `.channels`
**FIXED.** Both fields are `jsonb` columns storing raw JSON strings. Without
`@JdbcTypeCode(SqlTypes.JSON)`, Hibernate attempts to bind the default `"{}"` /
`"[\"EMAIL\"]"` values as `character varying` against a `jsonb` column — same
PSQL type error that broke `MessageEntity.attachments`. Added `@JdbcTypeCode` to
both fields. Tasks 2-8 and 2-9 will read these columns; fixing now prevents
insert failures in those tasks.

### C — `thread.setLastMessageAt(...)` persistence
**NOT A BUG.** `MessageThreadService.findOrCreateForClient` uses default
`@Transactional(REQUIRED)` propagation — it joins the outer transaction started
by `sendManual`/`sendForTrigger`. The thread entity is therefore managed within
the same `EntityManager` session. `threads.save(thread)` (JPA merge) correctly
persists the `lastMessageAt` update when the outer transaction commits. Test 2
(`manualSendBumpsThreadLastMessageAt`) reads back via a separate repository call
and passes green, confirming DB-level persistence. No fix needed.

### D — Null recipient goes to FAILED without a pre-check WARN
**FIXED (advisory severity).** Added an explicit `log.warn(...)` before constructing
`OutboundMessage` when `recipient` is null or blank. The WARN fires before the
`OutboundMessage` constructor throws `IllegalArgumentException`, making the root
cause immediately visible in logs without requiring exception message parsing.
The existing FAILED INFO log still fires as the final word per call.

### E — `@Audited` produces two audit paths, not one `MessageRouter#send`
**DOCUMENTED (Option 4).** Spring AOP cannot intercept self-invocations or private
methods — moving `@Audited` to `send(...)` (private) would produce zero audit rows.
Decision: leave `@Audited` on both public entry points. Audit rows are produced with
paths `MessageRouter#sendManual` and `MessageRouter#sendForTrigger`. Added javadoc
to both methods explicitly stating that task 2-10's `MessageSentTimelineHandler`
must match BOTH paths. No code-path change; the curator registration in 2-10 is
the implementation site.
