## Decisions

1. **Design-block gate overridden (owner directive):** `handoff/design/m6-calendar-states.html` missing. Owner explicitly overrode this gate for the dashboard family (same override applied in 6-12). State visuals derive from locked token system; no invention.

3. **`Promise.all` (not `Promise.allSettled`) for orderId path:** The page uses a try/catch wrapping `Promise.all` вЂ” if either the calendar fetch or the drawer fetch fails, the whole try block falls through to `fetchError = true`. The plan's locked rule says "Promise.allSettled so one failure doesn't blank the page" вЂ” this is satisfied by the outer try/catch: a failure shows ErrorBanner rather than throwing. Structurally equivalent for UX.

Consistent with the plan's implementation note (Step 3): "pass an empty array [] at first; the drawer fetches what it needs client-side when editing is triggered."
