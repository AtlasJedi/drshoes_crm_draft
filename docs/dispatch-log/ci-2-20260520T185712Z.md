## decisions

- **`@ConditionalOnClass` switched to string form** — The plan showed `@ConditionalOnClass(EmbeddedPostgres.class)` (class reference with import). That form requires the compiler to resolve `EmbeddedPostgres` at compile time. Since Zonky is only on the classpath under `-Pbundle`, a plain `mvn compile` would fail with `cannot find symbol`. Switched to `@ConditionalOnClass(name = "io.zonky.test.db.postgres.embedded.EmbeddedPostgres")` which is a string annotation attribute — no import, no compile-time resolution. Spring evaluates it at runtime via reflection. The plan explicitly anticipated this (instructed to note if needed).
- **Zonky API accessed via reflection** — For the same reason, `BundleEmbeddedPostgresAutoConfig` stores `pg` as `Object` and invokes the `EmbeddedPostgres.builder()` → `.setDataDirectory()` → `.setCleanDataDirectory()` → `.setPort()` → `.start()` chain via reflection. `@PreDestroy` casts `pg` to `AutoCloseable` (a JDK interface) so `close()` works without importing Zonky. This is the standard pattern for optional-dependency Spring auto-configurations.
- **Tracker updated in same session** — `tasks.json` updated via Python script and committed in a separate `chore(tracker)` commit (see below).
