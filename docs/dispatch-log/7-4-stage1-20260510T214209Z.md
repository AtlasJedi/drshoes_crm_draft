## Decisions / Surprises

- **MessageThreadEntity uses `OffsetDateTime`** not `Instant` for `discardedAt`. Plan test code passed `Instant.now()` which caused a compile error. Fixed `seedThread` helper to use `OffsetDateTime.now()` and added channel param.
- **`message_thread_unique_matched` partial index** (from V013): unique on `(channel, client_id)` WHERE `discarded_at IS NULL`. The happy-path IT test originally seeded 3 threads with the same client+channel, causing a constraint violation on the second non-discarded thread. Fixed by making `seedThread` accept a `channel` param and using `SMS` for the read thread (unreadCount=0).
- **`receivedAt` is nullable** in Order entity. Plan test code didn't set it in `seedOrder`, which would return null from `findLastOrderCreatedAtByClientId` and fail the `isNotEmpty()` assertion. Added `o.setReceivedAt(Instant.now())` in the helper.
- **`Order.code` is NOT NULL unique**: seeded with `"ORD-" + random 8-char UUID substring` to avoid collisions.
- `countByClientIdAndUnreadCountGreaterThan` (existing method without discard filter) is preserved; new method adds discard filter for ClientSummaryService. No existing callers changed.
- Commit SHA: feb75b3
