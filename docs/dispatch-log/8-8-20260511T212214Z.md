## Decisions

1. **Defensive guard on factory empty lists:** Plan's `run()` calls `threadFactory.createSampleThread(clients.get(0), orders.get(0))` unconditionally. With stub factories returning empty lists this would throw `IndexOutOfBoundsException` at app startup, breaking the test. Added `if (!clients.isEmpty() && !orders.isEmpty())` guard. This matches intended runtime semantics: when no clients/orders were created (already-seeded path or stub), no thread creation is attempted.

2. **@ActiveProfiles({"test","local"}) in test:** Plan's errata note anticipated this. `AbstractIntegrationTest` only activates `@ActiveProfiles("test")`; `DemoSeedRunner` requires `@Profile("local")`. Added `"local"` as the errata note directed. The `@ConditionalOnProperty` is satisfied by `@TestPropertySource`.

3. **No mvnw — used system mvn:** Project has no `mvnw` at root; `mvn` from Homebrew used. Same as 8-7 dispatch pattern.

4. **Module path is `backend/app` not `backend/apps/admin-monolith`:** Dispatch prompt uses the older naming convention. The actual Maven module is `backend/app`. Corrected at execution time.
