## Decisions / deviations from plan

### Items inline-create: INCLUDED
`CreateOrderRequest.java` includes `List<CreateOrderItemRequest> items` — inline items at create-time
ARE supported by the backend. The items section is included in the form.

Items are OPTIONAL (can be empty and added later via drawer 1-16). The form shows
"Brak pozycji — możesz dodać później w zleceniu." when no items are added, matching the
plan instruction.

### PLN price input: numeric string → cents at submit
Item price input is `type="number"` in PLN (decimal). `plnToCents()` parses the value,
strips commas, and does `Math.round(n * 100)`. Empty/invalid → 0 cents.

### `plannedPickupAt` date conversion
`<input type="date">` yields `yyyy-mm-dd`. Converted with `new Date(value + "T12:00:00").toISOString()`
(noon local time) to avoid TZ off-by-one at midnight.

### `source` field
`CreateOrderRequest.java` includes `OrderSource source`. Sent as `"ADMIN"` on every submit
from this form (per locked decisions: admin panel always uses ADMIN source).

### `receivedAt` field
`CreateOrderRequest.java` also has `receivedAt: Instant`. Omitted from the form — the backend
should default it to `now()`. Not wiring a date picker for this field; it is an implementation
detail surfaced in the drawer later if needed.

### Item row extraction
`NewOrderForm.tsx` is 116 LOC with the item row inlined. Extracting `NewOrderItemRow.tsx`
(62 LOC) keeps both under 120 LOC per granularity rule.

### `as any` cast cleanup
`orders/page.tsx` previously used `href={newHref as any}` for the `/admin/orders/new` Link
(route didn't exist at typecheck time in 1-14). Now that the route is built and typed routes
recompile, both casts were removed cleanly with 0 typecheck errors.

## Concerns
- `receivedAt` is not surfaced in the form. Backend should default to `now()`. If it doesn't, orders will have null `receivedAt`. To be verified when the backend runs end-to-end.
- Item price input allows `0` — no minimum validation. Intentional: zero-price items may be valid for tracking purposes (e.g. consultations).
