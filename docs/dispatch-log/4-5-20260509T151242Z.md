## Decisions / deviations from plan

### Errata #6 verification — DIVERGED

**Plan said:** request field `idempotency_key` (e.g. `payload.put("idempotency_key", msg.idempotencyKey())`).

**Reality (verified against https://www.smsapi.pl/docs on 2026-05-09):** SMSAPI does NOT have an `idempotency_key` request field. The correct field is:
- `idx` — custom identifier (max 255 chars, alphanumeric) returned in delivery callbacks
- `check_idx=1` — optional dedup flag that prevents sending two messages with the same `idx` within 24 h

**Decision:** Changed `idempotency_key` → `idx` throughout PayloadMapper and both test files. The `check_idx` flag is task 4-6's responsibility (SmsApiSmsGateway wires it when `msg.idempotencyKey() != null`). Added mandatory `TODO(plan-errata #6)` comment in both mapper classes and in `SmsApiPayloadMapperTest.java`.

**Response shape:** Confirmed matches plan — `list[0].id` is the stable message ID, error envelope uses `error` (int) + `message` (string).

### Jackson-databind added to sms-gateway pom
`SmsApiResponseMapper` requires `ObjectMapper`. sms-gateway pom had no Jackson dependency. Added `jackson-databind` without version — covered by `spring-boot-dependencies` BOM in parent pom (mirrors what task 4-3 did for email-gateway).

### 4 tests per mapper instead of 2
Plan stubs noted "cases follow same pattern" for 2 additional cases each. Implemented all 4 cases per mapper (total 8) since the plan text described them fully. Raises total from plan's expected 4 to 8 tests. All green.

## Carry-forward notes for task 4-6 (SmsApiSmsGateway)

1. **`idx` field name** — PayloadMapper already emits `idx` (not `idempotency_key`). SmsApiSmsGateway should add `check_idx=1` to the request body when `msg.idempotencyKey() != null` to activate SMSAPI's 24-h dedup window.
2. **jackson-databind** already in sms-gateway pom — no need to add again.
3. **`SmsApiPayloadMapper.toPayload(msg, from)`** signature: takes the `from` sender string from `SmsApiProperties`. Gateway calls this and then serializes the returned `Map<String, Object>` to JSON via `ObjectMapper` (or `RestClient` with Jackson message converter).
4. **`SmsApiResponseMapper.fromResponse(httpStatus, body)`** signature: takes raw HTTP status int and raw body string. Gateway must capture the body even on non-200 responses (RestClient `onStatus` handler or manual body read) before passing here.
