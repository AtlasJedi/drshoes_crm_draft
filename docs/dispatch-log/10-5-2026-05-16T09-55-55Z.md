## Key Decisions

1. **HttpServletRequest attributes instead of ThreadLocal** — plan proposed ThreadLocal on AddOrderNoteRequest but the `clearMoveContext()` call after `return` in the controller is unreachable in Java. Request attributes auto-clean at request end, are available to the aspect in the same thread, and require no try/finally ceremony.

2. **AddOrderNoteRequest implements HasAuditNote only** (not HasAuditLocationDiff) — location diff goes via request attributes, not via the marker interface. HasAuditLocationDiff.java still created per plan Step 3 for future use.

3. **AuditLogWriter chain** — existing 9-param overload delegates to new 11-param overload with null,null. All existing callers unchanged, no regression risk.

4. **IT uses mockMvc() + loginAsOwner() pattern** — plan template used non-existent `mvc.perform() + ownerSession()`. Corrected to match AdminWebTestBase contract.

5. **AuditLogRepository.findOrderTimelineRows** used for audit row assertion — `findByParentEntityIdOrderByCreatedAtDesc` does not exist in the repo.
