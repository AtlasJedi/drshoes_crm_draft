## Decisions

1. **Reality delta — unified CalendarOrderDto:** The plan specified two separate types (`CalendarOrderDto` for scheduled, `UnscheduledOrderDto` for unscheduled). The actual backend `CalendarResponseDto.java` (6-7) uses a **single** `CalendarOrderDto` record for both lists, with `plannedPickupAt` null for unscheduled entries and `receivedAt` null for scheduled entries. Types adapted accordingly; plan's `UnscheduledOrderDto` shape not created. Documented in `types.ts` JSDoc.

2. **Typed routes cast for future pages:** `/admin/orders/calendar` and `/admin/orders/kanban` are not yet in Next.js `StaticRoutes` (pages ship in later tasks). Used `import type { Route } from "next"` + `as Route` cast in TABS array to satisfy typed-routes enforcement without disabling it globally.

3. **Test mock pattern:** Followed `lib/dashboard/api-server.test.ts` (vi.stubGlobal + makeResp helper + import-after-mock) rather than the plan's global-assign pattern. The dashboard pattern is cleaner (afterEach restoreAllMocks, no global pollution).

4. **TypeScript fix — `mock.calls[0]` possibly undefined:** Added `!` non-null assertions on `fetchMock.mock.calls[0]![0]` and `[1]` in test to satisfy strict noUncheckedIndexedAccess.
