## Decisions

1. **AlertDialog substitution**: `@radix-ui/react-alert-dialog` is not in package.json. Used `@radix-ui/react-dialog` with controlled `discardOpen` state. The discard button retains `bg-red-600 text-white` destructive styling per plan.

2. **`detail` prop shape**: `UnmatchedThreadPanel` accepts `{ detail: ThreadDetailDto; onResolved: () => void }` per errata #2. `SelectedThread` passes the full `detail` from `useThreadPoller` (already `ThreadDetailDto`) when `detail.thread.unmatched` is true.

3. **`ClientCreateModal` onCreate signature adaptation**: The modal's `onCreate` is `(client: ClientDto) => void` (synchronous). The `assignUnmatched` call after creation is handled in `handleCreateAndAssign` (an async function in UnmatchedThreadPanel), not inside `onCreate` itself. This keeps ClientCreateModal's internal reset/close flow intact while still chaining assign → resolve.

4. **`handleCreateAndAssign` closes modal before assign**: `setCreateOpen(false)` is called first, then `assignUnmatched` is awaited. This gives immediate UI feedback (modal closes) even if the assign API call is slow.
