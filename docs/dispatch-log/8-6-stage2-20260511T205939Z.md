## Design Decision: Option B ‚Äî span ERROR on soft gateway failure

**The tension:** Plan acceptance criterion (b) at line 928 states unambiguously: *"on gateway exception the span status is `ERROR`"*. The test annotation at lines 1094-1097 hedged: *"status may be ERROR or UNSET ‚Äî assert channel at minimum."* Stage 1 chose Option A (span stays OK when gateway exception is caught internally).

**Decision: Option B is correct.** Rationale:

1. The plan's acceptance criterion is the authoritative spec; the test annotation was hedging the implementation ambiguity, not overriding the spec.
2. M8's owner directive is "extensive observability, OpenTelemetry would be amazing." A span with `StatusCode.OK` when a message delivery failed is actively misleading ‚Äî Jaeger's default error-span filter would miss all soft failures entirely, defeating the demo-grade observability goal.
3. The fix does NOT require rethrowing the exception (which would break existing dispatcher behavior). Instead, a `softFailurePredicate` is passed to `dispatchWithSpan`; the helper evaluates it after the action returns and sets ERROR if true.

**OTel SDK override behavior verified:** `SdkSpan.setStatus` bytecode confirms: once a span is `OK`, calling `setStatus` again is a no-op (lines 45-66 in bytecode). Therefore `Span.current().setStatus(ERROR)` from inside the lambda followed by `span.setStatus(OK)` in the helper would have been overwritten by OK. The predicate overload correctly gates the OK path.

## Decisions

1. **Predicate overload over `Span.current()` from lambda.** Setting `Span.current().setStatus(ERROR)` from inside the lambda then having `dispatchWithSpan` set `OK` afterwards would silently overwrite ERROR ‚Äî confirmed by SdkSpan bytecode (lines 45-66: OK‚Üíanything is a no-op, but ERROR‚ÜíOK is NOT protected). The predicate overload keeps all OTel logic in `MessagingSpanHelper` and keeps the dispatcher free of OTel API imports.

2. **Predicate overload keeps `MessagingSpanHelper` generic.** The no-arg-predicate overload delegates to the predicate overload with `t -> false`, preserving the simple API for callers that don't need soft-failure semantics.
