## Root Cause

`KanbanCardDto` (Java record and TS mirror) had no `version` field. The board wrapper fell back to `?? 0` for every card, so every drag-and-drop PATCH carried `version=0` against an order that had already been updated (real version=14+), yielding HTTP 409 from the optimistic-lock check.

## Decisions

- `Order.getVersion()` returns `int`, safe to widen to `long` in the DTO record — no boxing/migration needed.
- `KanbanCardDto` record in Java: version added at tail position (non-breaking for any positional test constructors; the integration tests use JSON assertions and `orderRepo.save()` helpers, not direct record construction).
- `KanbanControllerIntegrationTest` uses JSON path assertions only — no positional DTO construction — so no test changes were required there.
- Pre-existing tsc errors in `e2e/m11-fix-batch-sweep.spec.ts` and `lib/__tests__/locations.test.ts` are unrelated and were not introduced by this change.
