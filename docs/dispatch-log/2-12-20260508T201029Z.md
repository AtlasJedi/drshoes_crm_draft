## Decisions / deviations from plan

1. **Test adaptation — `AdminWebTestBase` real API vs plan skeleton**: Plan skeleton used
   `.with(loginAsOwner())` (treating `loginAsOwner()` as returning `RequestPostProcessor`).
   Real base class has `loginAsOwner()` returning void and `mockMvc()` as session-aware
   wrapper. Tests rewritten to match: `loginAsOwner()` call + `mockMvc().perform(...)`.
   Same adaptation as 2-11.

2. **Test isolation — `@AfterEach` reset**: Plan suggested flip-then-restore in a single test
   or `@AfterEach`. Used `@AfterEach resetTriggerState()` that sets `enabled=true` on all
   trigger rows. This ensures `toggleEnabledFlipsField` doesn't contaminate other tests
   running in the same shared Spring context.

3. **`TriggerDto.requiresManualConfirmation` accessor**: `TriggerEntity.isRequiresManualConfirmation()`
   is the boolean getter (not `getRequiresManualConfirmation()`). Used `.isRequiresManualConfirmation()`
   in `toDto()` to match the entity's Lombok-style naming. The DTO field is `Boolean` (boxed)
   per plan record signature; entity is `boolean` (primitive) — no mismatch at runtime.

4. **Structured logging in service and controller**: `TriggerService` logs
   `op=trigger.toggle outcome=ok id={} enabled={}` at INFO (on `setEnabled` only, per spec).
   `TriggersController` logs `op=trigger.list/get actor={} outcome=ok` plus
   `op=trigger.toggle actor={} triggerId={} enabled={} outcome=ok` for the PATCH endpoint.

5. **`MessagingExceptionAdvice` reuse**: No changes needed — already scoped to
   `com.drshoes.app.messaging.api`, so `TriggersController` is covered automatically.

6. **No `@Audited` on `list`/`get`**: `@Audited` is on `setEnabled` only, per spec.
   The `AuditLogAspect` still fires for all HTTP requests through its path-pattern curator.
