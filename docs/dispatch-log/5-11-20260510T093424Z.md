## Decisions / deviations from plan

1. **sendNewToClient added to MessageRouter**: Plan noted this method was to be added by 5-3 (slice-A) and was not yet present. Implemented here following the `sendReply` pattern — find-or-create thread by clientId+channel, build MessageEntity, delegate to dispatcher. No stub/TODO.

2. **Test base changed to AdminWebTestBase**: Plan template used `AbstractIntegrationTest` with a synthetic `owner()` post-processor via `SecurityMockMvcRequestPostProcessors.user(...)`. This fails because `message.sent_by` and `audit_log.actor_id` are FKs to `user_`. Switched to `AdminWebTestBase` (same decision as 5-10). All test calls use `.with(csrf())` as required.

3. **Audit log note**: The `AuditLogAspect` fires on the POST (HTTP method audit), logging 422 responses as status=500 in one audit row — this is a known aspect behavior for `ResponseStatusException` and matches existing pattern. Not a bug introduced here.
