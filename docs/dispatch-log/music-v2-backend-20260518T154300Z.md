## Deviations from plan

1. **@Audited has no `action=` attribute** — The plan specified `@Audited(action="MUSIC_PLAYLIST_CREATED", parent="#result.id")` but the actual `Audited` annotation only has `parent()`. Used `@Audited(parent=...)` matching all other service implementations in the codebase. Action is derived by AuditLogAspect from the method name, not a custom string.

2. **`PlaylistDto.summary()` fetches tracks size** — Since `MusicPlaylist.getTracks()` returns a lazy list, calling `.size()` in `summary()` within a transaction is safe. In unit tests (no JPA), the entity is constructed directly so this is a no-op on an empty ArrayList. No functional impact.

3. **`removeTrack` recompaction logic** — Implementation fetches all tracks from repo after deletion (including the deleted entity if the JPA flush hasn't happened). Used `remaining.remove(track)` to filter the deleted track from the list before recompacting. This is safe within the same transaction (delete is visible to subsequent queries). Test passes correctly because mock returns the full list including the deleted track.

4. **No `actor` parameter on `deletePlaylist`** — The plan didn't specify actor threading for delete, and `@Audited(parent="#id")` handles the audit row. Structured log uses the playlistId only. Consistent with other service methods that don't need actor in the service layer.
