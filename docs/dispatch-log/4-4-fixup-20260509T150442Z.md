# Task 4-4 fixup — stage-2 review findings applied

## Findings (from stage-2 review at b5a68d3)

1. **pom.xml — `spring-boot-starter-web` → `spring-web`.** Microlib must not pull a servlet container (Tomcat) into the runtime classpath. `RestClient` + `SimpleClientHttpRequestFactory` both live in `spring-web` alone. Version managed by Spring Boot BOM.
2. **`PostmarkAutoConfiguration` ordering** — add `before = EmailGatewayAutoConfiguration.class`. Without this, when both `messaging.email.provider=postmark` and the legacy `drshoes.email.provider=NOOP` (default) are present, the Logging gateway's `@ConditionalOnMissingBean` could fire before Postmark registers, producing `NoUniqueBeanDefinitionException`. The `before=` constraint forces Postmark to register first so the missing-bean guard sees it and skips the Logging fallback.

## Additional finding (discovered during verification)

3. **Test convention mismatch.** Stage-1 named the IT `PostmarkEmailGatewayIT.java`. The project's parent pom (`backend/pom.xml`) declares `maven-failsafe-plugin` in `<pluginManagement>` only — no `<execution>` block — so `*IT.java` classes are NEVER executed. The working convention in M2 is `*IntegrationTest.java` (Surefire matches `*Test`). Renamed to `PostmarkEmailGatewayIntegrationTest`.
