2. **Flake investigation**: Ran `AuditLogAspectIntegrationTest` once before starting. 6/0/0/0 — no flake observed. The plan noted the `AdminWebTestBase` FK-ordering flake from M5/M6. Based on prior sessions this was fixed in M6. No further action needed.

3. **`AuditSpanHelper` constructor is `public`** (not package-private): Test `AuditLogTraceIdIntegrationTest` instantiates `new AuditSpanHelper(otel)` directly (same package, but constructor visibility matches plan code block). Package-private would also work but public is the plan's code verbatim.

4. **No changes to `AuditLogWriter`**: The 8-param overload already existed (added in a prior Wave 1 task, verified at read time). No modification needed — carry-forward fact confirmed.

5. **`extractEntityType` placed in `AuditWriteCoordinator`** as `static` package-visible method (not private) so unit tests could reach it if needed. Plan had it as `private static` in `AuditLogAspect`; moved to coordinator as the natural owner. Made package-visible for testability.

6. **Span export errors in test log**: `HttpExporter` errors connecting to `localhost:4318` appear during full-suite run — this is expected (no OTel collector running in test env; OTLP exporter is wired for dev/prod). These are log-level ERROR but do not affect test pass/fail. Pre-existing behaviour from 8-4/8-5.
