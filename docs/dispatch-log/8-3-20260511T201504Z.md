## Decisions

- **No index added.** The plan explicitly deferred the index: trace_id lookups in this milestone are driven by Jaeger (which maintains its own index). A Postgres index on `audit_log.trace_id` would add write overhead with zero current query benefit. Deferred per plan.
- **varchar(32) length confirmed correct.** W3C traceparent trace-id is 128 bits encoded as 32 lowercase hex characters (no dashes). `varchar(32)` is the exact fit.
- **Nullable semantics.** Column is `NULL` â€” this is correct. Background jobs, startup hooks, and any call path without an active OTel span will not have a trace context. Existing write paths (5-param, 6-param, 7-param) all flow through the 7-param overload which now delegates to the 8-param overload with `null`.
- **No FK risks.** `trace_id` is a plain string column with no foreign key constraint. No referential integrity concerns.
- **Test isolation.** `AuditLogTraceIdMigrationIntegrationTest` extends `AbstractIntegrationTest` (Testcontainers Postgres with `withReuse(true)`). The test queries `information_schema.columns` which reflects the current schema state after Flyway runs at context startup. No inter-test state is mutated. Isolation is clean.
- **Naming convention.** Used `*IntegrationTest.java` suffix to match the project convention (`AuditLogAspectIntegrationTest`, `OrderControllerIntegrationTest`, etc.) and to ensure Surefire (not Failsafe) runs it â€” per the session memory note that `*IT.java` files never run under the current Failsafe-in-pluginManagement-only setup.
