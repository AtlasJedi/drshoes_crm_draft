## Decisions / deviations from plan

### Fix 1: `methodWithoutAuditedAnnotationLeavesParentEntityIdNull`
**Problem:** Called `clientService.create()` directly — no `RequestContextHolder` in scope,
so `persistHttp()` returned early (attrs == null guard), zero rows written, and the
`forEach` over an empty list asserted nothing. False green.

**Fix:** Switched test class from `AbstractIntegrationTest` to `AdminWebTestBase` (which adds
`@AutoConfigureMockMvc`). The test now uses `mockMvc()` to POST to `/api/admin/clients`
with a logged-in OWNER principal. This fires the real HTTP request context, the controller
audit pointcut writes exactly one row (status=201), and the assertion checks that row's
`parent_entity_id IS NULL`. If the aspect were changed to populate `parent_entity_id` on
HTTP rows, the test would correctly fail.

**Lifecycle reconciliation:** `AdminWebTestBase.seedUsers()` (`@BeforeEach`) seeds
`owner@test.pl` and `emp@test.pl` and clears clients+users. The audit test's own
`@BeforeEach seedAuditData()` runs after (JUnit 5: superclass before subclass) and
creates the `clientId` needed for order-based tests. `@AfterEach` ordering is reversed
(subclass first): `cleanupAuditData()` clears audit/order rows before `AdminWebTestBase`
cleans clients+users. No FK violations.

**Login tests:** `TestRestTemplate` remains `@Autowired` and works in the `RANDOM_PORT`
context that `AdminWebTestBase` inherits. Updated credential from `misza@drshoes.pl / p`
to `owner@test.pl / pass` (the user seeded by `AdminWebTestBase.seedUsers()`).

### Fix 2: `spelFailureDoesNotBreakAuditRow`
**Problem:** Tested `AuditedParentResolver.resolve()` in isolation, then tested the
happy-path `orderService.addItem` — never ran a method with a broken SpEL expression
through the full AOP proxy pipeline. The exception-safety claim was unproven end-to-end.

**Fix:** Added a `@TestConfiguration` inner class registering `BrokenSpelAuditedService`,
a `@Service` with `@Audited(parent="#nonExistentVariable.toString()")`. The `.toString()`
chain is critical: `#nonExistentVariable` alone returns `null` (undefined variable →
null in `SimpleEvaluationContext`, no exception), while `.toString()` on null forces
`SpelEvaluationException: EL1011E: Method call: Attempted to call method toString() on
null context object`, which triggers the `catch (Exception e)` block and the WARN log.

The test calls `brokenSpelAuditedService.doWork()` through the autowired Spring proxy.
Asserts:
- (a) method returns "sentinel" (no exception to caller)
- (b) `auditLog.count()` delta == 1
- (c) that row's `parent_entity_id IS NULL`
- (d) `AuditedParentResolver` logger emits ≥1 WARN containing `op=auditParentEvalFailed`
     and `#nonExistentVariable.toString()`, captured via Logback `ListAppender`
