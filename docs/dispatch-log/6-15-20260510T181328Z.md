## Decisions

1. **`UnscheduledOrderDto` does not exist** — the plan's test stub referenced `UnscheduledOrderDto` from `@/lib/calendar/types`, but that type was intentionally not created in 6-13 (noted in types.ts header comment). The unified `CalendarOrderDto` is used for both scheduled and unscheduled entries. Test adapted accordingly with `CalendarOrderDto` fixture.

2. **`router.push(...) as Route` cast** — Next.js typed router requires `RouteImpl<...>` for template-literal strings. Followed the exact pattern from `CalendarCell.tsx` (which already existed). Import `type { Route } from "next"` + cast on the push call.

3. **Test fix: `getByText` → `getAllByText`** — the plan's test used `screen.getByText("Maciek N.")` followed by `screen.getAllByText("Maciek N.")` in the "renders each row" test. With two rows rendered, `getByText` throws "found multiple elements". Removed the redundant single-match assertion; `getAllByText(...).toHaveLength(2)` is sufficient.

4. **Inline styles for design fidelity** — followed `admin.jsx:578-613` closely (inline `style={}` for pixel-level values matching the design JSX). Tailwind classes used only for layout utilities (`flex`, `text-left`, `gap-2`, etc.) where the design didn't specify exact px values. This matches the existing `CalendarMonthGrid.tsx` + `CalendarCell.tsx` pattern.

5. **Drag deferred** — drag-and-drop is not implemented (future milestone). The drag hint renders as a static `<p>` (disabled by intent, no `draggable` attribute or handlers). The icon placeholder `⠿` is `aria-hidden`.
