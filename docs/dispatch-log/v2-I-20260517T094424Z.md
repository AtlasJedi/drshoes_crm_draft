## Root Cause / Motivation

OrderDrawer's close handler previously hardcoded `/admin/orders` as the redirect target, breaking calendar and kanban close (redirects user away from those pages). Using `usePathname()` captures the current page at open-time and returns to it on close.

ClientOrdersRows extracts the clickable-row concern from a pure server component into a dedicated client component, enabling the same `?orderId=` drawer pattern used on the orders list, kanban, and calendar pages.

## Decisions

- `ClientOrdersRows` uses the full `/admin/clients/${clientId}/zlecenia` path for `router.push` (not `pathname`) to avoid stale closure issues with server-rendered pages.
- Cross-client guard: if `drawerOrder.clientId !== id`, `drawerOrder` is set to null and the drawer silently does not render.
- OrderDrawer.test.tsx fix is included here (not a separate commit) since the test breakage was introduced by this task's `usePathname` addition.
