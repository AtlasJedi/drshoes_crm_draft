## Decisions

1. **Dispatcher bypass:** Messages inserted directly via `MessageRepository.saveAll()` ‚Äî no `MessageRouter` invocation. This ensures no real provider sends (email/SMS/WhatsApp) fire during seed. The plan explicitly permits repository-direct insertion as the preferred path.

2. **Triggers off (n/a):** Because we bypass `MessageRouter` entirely, the `sendTriggers` flag is irrelevant. No trigger evaluation occurs at all during seed.

3. **Audit semantics:** `AuditLogAspect` pointcut matches `..api..` controllers only. Seed runs in an `ApplicationRunner` outside any HTTP request, so no audit rows are written. This is expected and correct for seed data ‚Äî noted in carry-forward facts.

4. **Idempotency mechanism:** `DemoSeedRunner.run()` checks `clientRepository.count() >= SEED_THRESHOLD (6)` and returns early if already seeded. `MessageThreadService.findOrCreateForClient()` is itself idempotent (find-or-create). On a second seed call the runner exits before reaching `threadFactory.createSampleThread()`.

5. **`@ActiveProfiles({"test","local"})` added to test:** The plan's test template omitted `@ActiveProfiles` but `@Profile("local")` beans (DemoClientFactory, DemoOrderFactory, DemoThreadFactory, DemoSeedRunner) require the "local" profile to load. Pattern follows `DemoSeedDataIntegrationTest`. Without this, Spring would fail to wire `DemoSeedRunner` as a bean.

6. **Channel:** EMAIL per plan acceptance criteria (4 messages all channel=EMAIL).

7. **`setSubject()` confirmed:** `MessageThreadEntity` has `setSubject(String)` at line 67 ‚Äî errata note in plan was a non-issue.

8. **Maven command correction:** The plan says `cd backend && mvn -pl app -am test`. Running from repo root (`mvn -pl backend/app -am test`) fails because there is no root `pom.xml`. Must `cd backend` first. Build command is correct as documented in carry-forwards.
