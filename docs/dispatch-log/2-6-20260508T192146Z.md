## Decisions / deviations from plan

**MessageThreadEntity constructor made public:**
All messaging entities use `protected` no-arg constructors (JPA minimum). `MessageThreadService`
is in `com.drshoes.app.messaging.service` — a different sub-package from the entity in
`com.drshoes.app.messaging.domain`. Java `protected` is not accessible across sub-packages
(only within the same package or via inheritance). Changed the no-arg constructor from
`protected` to `public` in `MessageThreadEntity`. This is safe — JPA accepts either visibility.

**Test helper — local JDBC pattern (not base class):**
`AbstractIntegrationTest` has no client-creation helper. Per task instructions, added a local
`createClientAndReturnId()` method using `JdbcTemplate` to INSERT a minimal `client` row
(id, first_name, phone='+48 600 000 000' satisfying `client_contact_present` CHECK). Uses
`@AfterEach` cleanup iterating a `List<UUID> insertedClientIds` to delete `message_thread`
rows (FK) then `client` rows. Pattern is consistent with 2-4's `createMinimalOrder()`.

**repo.count() assertion removed:**
The plan stub used `assertThat(repo.count()).isEqualTo(1)` but this is fragile when tests run
in any order (another test may have inserted threads). Replaced with
`assertThat(repo.findFirstByClientIdOrderByCreatedAtAsc(clientId)).isPresent()` which verifies
idempotency without relying on global state. The core assertion (`first.getId() == second.getId()`)
is retained.
