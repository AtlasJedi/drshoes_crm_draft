## Change
The composer's form body (textarea + location select + "dodaj wpis" submit) is now hidden
by default. The section header "Dodaj wpis do historii" was previously a `<div>`; it is now
a `<button type="button" aria-expanded={open}>` with a `▾`/`▸` chevron, toggling local state
`open`. After a successful submit, the form collapses and resets (`text=""`, `target` back
to `currentLocation`).

Side cleanup: `err: any` → `err: unknown` with a typed narrow.

Existing cases updated to click the toggle first (composer no longer always-visible).

## Decisions
- Toggle = native button styled to keep mono caps header look, not Radix Collapsible (overkill).
- Reset both `text` AND `target` on submit so re-opening shows a clean slate, not stale text.
