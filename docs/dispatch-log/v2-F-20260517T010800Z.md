## Key Decisions

- `DONE` kind implemented via V027 migration adding `target_status` column to `audit_log`. `OrderController.changeStatus` sets request attribute `audit.targetStatus`; `AuditWriteCoordinator` reads and passes to writer; `TimelineEventCurator` emits `DONE` when value is `"WYDANE"`. No other approach avoids schema changes while giving curator access to the target status.
- `OrderDrawerPhotos` rewritten to use inline `uploadPhoto` call (drops `PhotoUploader` expand/collapse panel). `onRelabel` in `PhotoLightbox` simplified to `async () => { await refresh(); }` since full relabel flow wasn't needed in the new component.
- `OrderDrawerStatusGrid` CSS uses inline styles matching handoff tokens directly (no new CSS module) — Tailwind lacks the `::before` paint-fill animation. The paint fill animation is represented by solid fill on the active state; full CSS animation would require a CSS module or global styles.
- Stepper merged into `OrderDrawerHeader` — eliminates `OrderDrawerStatusTimeline` which was a separate component below the header.
- All 5 action handlers preserved: status change (StatusGrid → StatusChangeTriggerDialog), photo upload (inline tile), note add (NoteComposer unchanged), opis save (OrderDrawerOpis onBlur), mark-wydane button (footer).
