## Behavioural delta
Before: `OrderDrawerStatusChanger.handleConfirm(note)` discarded the dialog's `sendTriggers` flag. Both "Tylko zmień status" and "Wyślij wiadomość" sent the same payload; the backend defaulted `sendTriggers` to `TRUE` in the compact ctor, so every status change fired the trigger.

After: the flag is forwarded end-to-end. "Tylko zmień status" → `sendTriggers=false` → backend skips trigger. "Wyślij wiadomość" → `sendTriggers=true` → trigger fires.
