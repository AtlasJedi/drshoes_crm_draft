## Decisions

1. **Testing infrastructure upgrade:** The plan required `@testing-library/react` component tests but the existing `vitest.config.ts` used `environment: "node"`. Upgraded to `jsdom` + `globals: true` + `vitest.setup.ts` with `@testing-library/jest-dom`. Existing api-server tests continued to pass in jsdom environment (they only use vi.mock + fetch stubs, no node-only APIs).

2. **MixDonut arc rotation — reduce instead of let:** The plan's code used a `let rotationDeg` mutation inside `map`, which triggered `react-hooks/immutability` lint error. Fixed by replacing with a `reduce` that accumulates `{ arcs, deg }` without any mutable variable. Identical runtime behavior; same visual output.

3. **No charting library:** The plan explicitly uses raw SVG, not a third-party chart library. No new chart lib dep added.

4. **No `"use client"`:** All three components are pure server components. No interactivity needed for the presentational layer — data flows in as props from the server page (6-12).

5. **KpiTilesRow revenue label:** Uses `new Date().toLocaleString("pl-PL", { month: "long" })` for the month name (e.g. "maj") matching the design's "Przychód · maj". This runs server-side at render time.

6. **Test count:** Plan stated "14 vitest cases" but the spec code blocks sum to 3+5+5=13. Implemented exactly what the spec defined; all 13 pass (18 total including 5 pre-existing api-server tests).
