## Decisions / deviations from plan

### Decision 1 — `isNetworkError()` helper needed (same as PostmarkEmailGateway)

**Problem:** Plan's GREEN code catches only `ResourceAccessException`. First GREEN run: retry
test (Test 4) failed with `RestClientException: Error while extracting response for type
[java.lang.String] and content type [application/octet-stream]` — `SocketException: Connection
reset` thrown during `getHeaders()` (header-read phase, before status code available), wrapped
as bare `RestClientException`, NOT `ResourceAccessException`.

**Fix:** Added `isNetworkError(RestClientException e)` returning true for
`ResourceAccessException` OR any `RestClientException` whose cause chain contains an
`IOException`. `executeWithRetry` catches `RestClientException`, calls `isNetworkError()`, and
re-throws if it's not a network error. This exactly mirrors `PostmarkEmailGateway` (4-4
dispatch log, Decision #2). Reviewer should verify that `HttpClientErrorException` /
`HttpServerErrorException` (4xx/5xx) do NOT trigger this: they extend
`RestClientResponseException` which extends `RestClientException`; their cause chain wraps
an `HttpMessageConversionException` or similar, containing no `IOException`. Furthermore,
the `onStatus` lambda suppresses the throw before `RestClientResponseException` is raised at
all, so 4xx/5xx flows never enter the retry path.

### Decision 2 — `check_idx` placed in gateway, not mapper

Carry-forward rule #3 allows placement in the mapper signature or in the gateway. Chose to
add it in `gateway.send()` via a `LinkedHashMap` copy of the mapper's result, keeping the
mapper a pure `OutboundMessage → Map` function. No mapper signature change. The `TODO
(plan-errata #6)` comment in `SmsApiPayloadMapper` already notes that `check_idx` is task
4-6's responsibility.

### Decision 3 — `SmsApiSmsGateway` is 135 LOC (over 120 cap)

Added fields: `isNetworkError()` helper adds ~15 lines. Same situation as
`PostmarkEmailGateway` (4-4: 157 LOC). No clean split: all methods share `restClient`,
`props`; `isNetworkError()` is tightly coupled to `executeWithRetry()`. Flagged for reviewer.
A `SmsApiNetworkSender` extract holding `doPost` + `executeWithRetry` + `isNetworkError` is
one option if reviewer considers it warranted.

### Decision 4 — `SimpleClientHttpRequestFactory` in test setUp

`RestClient.builder().baseUrl(...)` without a factory defaults to JDK `HttpClient` which
prefers HTTP/2. WireMock standalone (Jetty 11) does not support HTTP/2, causing test failures
on RST_STREAM. Set `SimpleClientHttpRequestFactory` (HTTP/1.1) in test setUp. Same pattern
as PostmarkEmailGatewayIT (4-4 dispatch log, Decision #6). Production autoconfig also uses
`SimpleClientHttpRequestFactory`.
