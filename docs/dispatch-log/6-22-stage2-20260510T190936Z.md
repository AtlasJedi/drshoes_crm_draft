## Review Findings

### 1. `sendTriggers` wiring vs 6-9 backend contract

`BulkActionBar.tsx` initializes `sendTriggers` state as `false` (line 27) and wires it to the checkbox via `onChange={(e) => setSendTriggers(e.target.checked)}` (line 99). `handleSubmit` passes `sendTriggers` directly into `bulkChangeStatus({ ..., sendTriggers })` (line 45). `bulk-api.ts` serialises it as JSON into the POST body. The backend 6-9 contract uses boxed `Boolean` + compact ctor normalization defaulting to `TRUE` for absent field; here the field is always explicitly sent (`false` default, checkbox opts in to `true`). This is correct — explicit `false` propagates cleanly; checked checkbox sends `true`. The `BulkActionBar.test.tsx` fifth test (`Wykonaj calls bulkChangeStatus`) asserts `sendTriggers: false` for the unchecked default. **CLOSED — wiring correct.**

### 2. `onClear` semantics on partial success

`handleSubmit` calls `onClear()` only when `res.failed.length === 0` (full success). On partial failure, `onClear` is not called; the modal opens and the user must acknowledge. `handleResultClose` always calls `onClear()` regardless of outcome. On full success, `onClear` is therefore called twice: once in `handleSubmit`, once in `handleResultClose`. The double-call is safe — `setSelected(new Set())` on an already-empty `Set` is idempotent. The plan's Stage 2 checklist (line 9982) specifies: "confirm `onClear()` is NOT called when `result.failed.length > 0`" — this holds. **CLOSED — semantics correct, double-clear is idempotent.**

### 3. `BulkResultModal` raw status display — NEEDS-FIXUP (applied)

Stage 1 rendered `{row.fromStatus} → {row.toStatus}` as raw enum strings (e.g. `W_REALIZACJI`) for the success-row transition display. The plan's Step 8 reference code explicitly showed `STATUS_LABELS_PL[row.fromStatus as OrderStatus]`. The locked convention "all UI copy in Polish" is non-negotiable. `STATUS_LABELS_PL` is already exported from `apps/web/lib/orders/status.ts`. Fixup applied: imported `STATUS_LABELS_PL` into `BulkResultModal.tsx` and replaced raw `row.fromStatus`/`row.toStatus` with `STATUS_LABELS_PL[row.fromStatus]`/`STATUS_LABELS_PL[row.toStatus]` (types are already `OrderStatus` — no cast needed). The test assertion `getByText(/W_REALIZACJI/i)` was testing the raw string (accidental contract); updated to `getByText(/W realizacji/i)` to assert the Polish label. **FIXED — commit `6044f49`.**
