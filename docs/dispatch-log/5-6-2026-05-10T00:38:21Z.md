## Decisions

1. **Allowlist property**: `messaging.sms.smsapi.allowlist` — distinct from M4's `callback-allowlist`
   as specified in plan line 2388. Allows delivery callback and MO inbound allowlists to diverge.
2. **IP header hardcoded**: `Cf-Connecting-Ip` — not configurable (MO inbound always behind Cloudflare;
   no configurable-header indirection needed unlike the M4 outbound GET controller).
3. **`Set<String>` instead of `List<String>`**: allowlist injected as CSV string, split into Set for
   O(1) contains check. Consistent with the controller's fail-closed semantics.
4. **`DataAccessException` catch**: race-window safety net (same pattern as PostmarkInboundController 5-5).
   Returns 200 + duplicate=true with null IDs. No dedicated IT case (race window not reproducible
   deterministically in single-threaded test).
5. **DTO expansion callsite fix**: `InboundMessageServiceTest.smsPayload()` helper updated to pass
   null for `smsTo` (3rd positional arg). `InboundMessageService.buildSmsMessage()` did not reference
   `smsTo` — no service-layer change needed.
6. **`@Audited(parent="#result.threadId")`** already wired on `InboundMessageService.recordSmsInbound()`
   from task 5-4; controller delegates cleanly, no additional @Audited annotation needed on controller.
