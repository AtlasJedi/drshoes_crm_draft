## Decisions

- New fields land at positions 13/14 (between `clientName` and `discardedAt`) so the existing positional mapper diffs are minimal and the contact triple stays adjacent.
- `loadClients` now caches the full `Client` entity (Map) instead of pre-projecting to a `String` name. Required because the mapper needs three fields off the same client; doing the lookup once per list still N=1 query (`findAllById`).
- `clientEmail` / `clientPhone` are **nullable** (matching the entity columns). Composer must guard for both being null (rare: client with no contact, e.g. seeded fixture).
- WHATSAPP enum value is intentionally not surfaced in the composer toggle — out of M5 scope; channels in the toggle are restricted to `["EMAIL", "SMS"] as const` per plan.
