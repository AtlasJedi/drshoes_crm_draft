## Decisions

1. **V017 not V016** — V016 was already taken by `V016__order_quote_and_advance.sql` (M8 task). Plan header said V016 but reality is V017. Deviation from plan, documented here.

2. **Backend package `com.drshoes.app.sklep`** — plan draft showed `com.drshoes.app.sklep` as the package. Used flat package (no sub-packages) since the feature has only 5 classes, all cohesive.

3. **Controller split for listLatest** — plan specified a `GET /api/admin/sklep/reservations?limit=N` endpoint across all products (for FreshReservationsPanel). To avoid Spring path-variable ambiguity between `/{productId}/reservations` and `/reservations`, both are under the same `@RequestMapping("/api/admin/sklep")` controller with distinct `@GetMapping` paths — Spring resolves this correctly by literal vs template matching.

4. **FreshReservationsPanel fallback** — component now fetches from real API but gracefully falls back to static PLACEHOLDER_ROWS on error (not empty). This preserves the dashboard's visual completeness during demo even if the DB has no reservations seeded.

5. **OXC parse error fix** — Polish `„` opening guillemet (U+201E) inside double-quoted JS string in the test caused OXC/vite transform failure. Fixed by switching to regex matcher `/może wpaść w czwartek/`.

6. **Text node split fix** — `{i + 1}. {r.clientName}` renders as multiple text nodes in one `<span>`, so `getByText("Karol Jastrzębski")` fails with exact match. Fixed all name assertions to use regex `/Karol Jastrzębski/`.

7. **`AdminWebTestBase` for IT** — used `AdminWebTestBase` (not bare `AbstractIntegrationTest`) since it provides `loginAsOwner()` / `loginAsEmployee()` + MockMvc session injection. The `product_reservation` table has no FK to the `user_` or `client` tables so the base teardown doesn't need modification.
