## Decisions

### Schema mismatch — currentStorageLocationId not synced

**Finding:** `storage_location.id` is `BIGSERIAL` (Long), introduced in V018. `order_.current_storage_location_id` is a legacy `UUID` column from V001 whose FK was dropped by V018's `DROP TABLE ... CASCADE`. The two types are incompatible — a Long id cannot be stored in a UUID column without a migration.

**Decision:** Skipped the `currentStorageLocationId` UUID column sync. The frontend TypeScript type (`lib/types.ts`) only declares `location?: string` (varchar name) — the UUID column is unused by the frontend. The denormalized `location` string remains the sole source of truth for display.

**Recommendation for owner:** Either (a) add a migration to drop `current_storage_location_id` UUID and add `current_storage_location_id BIGINT REFERENCES storage_location(id)` with a proper FK, or (b) keep relying on the `location` varchar name only. The plan's `findIdByNameAndActiveTrue` returning `Optional<UUID>` cannot be implemented without a schema migration.

**Plan step 3** (add `findIdByNameAndActiveTrue`) — not implemented due to type mismatch. `StorageLocationRepository` already has `existsByNameAndActiveTrue(String)` which is sufficient for the existing validation guard.

### quotedPriceCents removal from PATCH

The `OrderUpdateDiff.computePolish` still has a branch that references `req.quotedPriceCents()` for audit diff generation. Since the rejection throw in `OrderService.update` happens AFTER `computePolish` is called (the diff is computed before mutations), the diff branch is dead code. It was left in place to avoid churn — it can be cleaned up in a future pass if desired.
