## Decisions

1. **New columns show "—"** — `OrderListRow` projection (backend) does not include `receivedAt`, `createdAt`, or `pickedUpAt`. Sort is purely server-side ordering. Adding columns with "—" data cells is consistent with the existing "Wykonawca" column and makes the sort UI discoverable. No backend projection change needed.

2. **`SortableColumnHeader` is a standalone `"use client"` component** — `OrdersTable` is already `"use client"`, so no additional client boundary needed. The component is extracted to its own file for testability and to keep `OrdersTable` under the 80 LOC soft cap.

3. **`parseSortParam` exported** — allows direct unit testing without rendering the component. Clean utility boundary.

4. **LoginForm.tsx lint fix (out-of-plan)** — pre-existing `AnyRoute` unused type alias and `useRouter` unused import were blocking `--max-warnings=0`. Removed as an inline hygiene fix per dispatch protocol §inline-trivial-fixups. Documented here.

5. **Clicking a different column always resets to DESC** — matches the spec: "clicking a different column resets to that column DESC." Clicking the active column toggles direction.

6. **Page param reset on sort change** — `params.delete("page")` on every sort click so changing sort always shows page 0. Prevents stale pagination.
