## Concerns / flags for subsequent tasks

1. **`application-local.yaml` `demo.seed.enabled: true` is hardcoded.** Per plan, local profile enables demo seed unconditionally. The env var `DRSHOES_DEMO_SEED_ENABLED` in `application.yaml` provides the override mechanism for non-local environments. This is the intended design.
2. **Jaeger endpoint `http://jaeger:4318` only resolves inside docker-compose network.** For native local dev outside compose, the default `http://localhost:4318` (from `application.yaml`) applies. When the `local` profile is active outside compose, OTLP export will fail silently (SDK NOOP fallback). This is expected until task 8-9 wires docker-compose.
