## Decisions

1. **BSD grep `\|` in BRE** — On macOS, `grep -v '^\| Feature'` treats `\|` as alternation in BRE, causing the pattern to match everything (empty match via `^` arm). Fixed by adding `-E` flag to all `grep -v` calls inside `extract_rows()`: `grep -vE '^\| Feature'` and `grep -vE '^\|---'`. This is not in the plan's script verbatim but is required for macOS BSD grep correctness. Documented here as a macOS-compat fix.
2. **awk field numbering verified** — With `awk -F'|'`, table rows starting with `|` yield `$1=""`, `$2=Feature`, `$3=URL`, `$4=Frontend`, `$5=Backend`, `$6=Tests`. Plan's field assignments are correct. Filter uses `$2` for Feature keyword match; path extraction uses `$4;$5;$6`.
