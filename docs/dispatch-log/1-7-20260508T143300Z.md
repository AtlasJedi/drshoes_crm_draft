## Decisions / deviations from plan

1. **`OrderService.list()` takes a single `OrderStatus` (not `List<OrderStatus>`).** The service
   and `OrderSpecifications.forList()` both accept a single status value. The controller exposes
   a single `?status=` query param matching the service signature. The plan said
   `List<OrderStatus>` — accepted deviation since the service is already shipped (1-6 is locked).
   Supporting multi-status would require a service + spec change that is out of scope for 1-7.

2. **FK cleanup in test `@AfterEach`.** `AdminWebTestBase.cleanupUsers()` calls
   `clients.deleteAll()`, but `order_.client_id` FK prevents that while orders exist.
   Added `@AfterEach cleanupOrders()` in the test subclass (JUnit 5 guarantees subclass
   after-each runs before superclass), deleting `order_item` then `order_` first.

3. **Controller LOC: 142** — exceeds 120 LOC ceiling. CLAUDE.md permits CRUD controllers as an
   explicit exception (same as JPA entities). All 9 endpoints in one class; comment documents this.

4. **`RbacService.canDeleteOrders` already existed** (OWNER-only, shipping from 0B). No change needed.
   Bean name is `@Component("rbac")` — `@PreAuthorize("@rbac.canDeleteOrders(authentication)")` used.

5. **`OrderExceptionHandler` scoped to `OrderController`** via
   `@RestControllerAdvice(assignableTypes = OrderController.class)` to avoid
   bleeding into `ClientExceptionHandler`'s domain.
