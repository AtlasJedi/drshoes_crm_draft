## Decisions / deviations from plan

**1. delivered_at does NOT exist on order_ — schema reality errata was wrong.**
The dispatch input said "order_.delivered_at TIMESTAMPTZ exists (V001 line 270)". Line 270 of V001 is inside the `message` table, not `order_`. Added V007 migration to add `delivered_at TIMESTAMPTZ` to `order_` with a sparse index. This is the correct handover-timestamp column for AFTER_HANDOVER_Y_DAYS.

**2. Native query chosen over HQL CAST for Instant→LocalDate.**
HQL `CAST(o.plannedPickupAt AS LocalDate)` behaviour varies across Hibernate versions when the underlying column is `TIMESTAMPTZ` and the Java field is `Instant`. Used native query with Postgres `(col AT TIME ZONE 'Europe/Warsaw')::date = :targetDate` instead. Deterministic and timezone-correct.

**3. @EnableScheduling added to DrShoesApplication.java.**
Plan referenced `Application.java`; actual entry class is `DrShoesApplication.java`. `@EnableScheduling` added alongside `@SpringBootApplication`.

**4. Clock injection via MessagingClockConfig @Bean.**
Tests use the real system Clock (no override needed) because target dates are computed dynamically with `LocalDate.now(PL).plusDays(N)` matching exactly the day-offset the job uses. No test-time Clock mock required.

**5. Test cleanup: message_thread has no order_id column.**
Cleanup initially tried `DELETE FROM message_thread WHERE order_id = ?` — that column does not exist (message_thread only has client_id). Fixed to delete messages and trigger_fires by order_id, then message_threads by client_id.
