## Decisions / deviations from plan
- **Three-file split instead of two.** Plan anticipated `OrderDrawerItems.tsx` + optional `ItemEditRow.tsx`. The display row logic (pill, price, edit/remove buttons, delete confirm) was extracted into `OrderItemRow.tsx` to keep all files under 80 LOC. `OrderDrawerItems.tsx` ended at 83 LOC — 3 lines over the aspirational cap after splitting three ways; no further meaningful split was possible.
- **`getOrder` refetch after every mutation.** `addOrderItem` / `updateOrderItem` return `OrderItemDto` only (not the full `OrderDto`), so `getOrder(order.id)` is called after each successful mutation to refresh `totalPriceCents` and the item list. This matches the spec requirement exactly.
- **No `expectedVersion` on item endpoints.** Backend `CreateOrderItemRequest` / `UpdateOrderItemRequest` have no version field — items don't carry `@Version`. Only the parent `Order` entity uses optimistic locking. The 409 conflict path is still wired in the catch block to handle any unexpected conflicts.
- **`pricePLN` formatter inlined in `OrderItemRow.tsx`** as `fmtPLN` (exported for reuse) — no shared formatter existed in `lib/`. The `OrdersTable.tsx` uses its own inline private version. Consistent with project pattern; extract to `lib/orders/format.ts` in M2 if needed.
- **`run` helper** centralises setBusy/setConflict/log pattern for all three mutation handlers, reducing repetition by ~15 lines.

## Logging / metadata wired
- `createLogger("order-items")` in `OrderDrawerItems.tsx`

- `op=addItem outcome=conflict` logged at WARN on 409

## Concerns
- `OrderDrawerItems.tsx` is 83 LOC (3 over the 80 LOC cap). Already split into 3 files — no further split without triviality. Document as acceptable overage.
- After add/edit/remove, `getOrder` refetches the full order (a second round-trip). For M1 this is fine; in M2 consider returning updated `OrderDto` from item mutation endpoints to avoid the extra fetch.
- Delete confirm is inline (not a modal). This matches the "inline preferred" directive in the dispatch prompt. No Radix Dialog added.
