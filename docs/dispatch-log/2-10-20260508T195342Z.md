## Decisions / deviations from plan

### Curator extension API
`TimelineEventCurator` is now constructor-injected with `MessageSentTimelineHandler`.
Dispatch is the first call inside `curateInternal` (before any M1 item pattern checks),
matching the plan's "BEFORE M1 ORDER_*/ITEM_*/STATUS_CHANGED branch logic" requirement.
The handler returns `null` for non-matches; curator wraps non-null result in `Optional.of`.

### Dual-path matching
`MessageSentTimelineHandler.toEvent()` checks `row.getPath()` against the two string
constants `"MessageRouter#sendManual"` and `"MessageRouter#sendForTrigger"` using
`String.equals()` (not Pattern — paths are exact, no wildcards needed).
Both paths are covered by Test 2 (`triggerFiredMessageAlsoEmitsTimelineEvent`).

### HTTP row skip
Added `MESSAGES_SEGMENT_PATTERN` to `TimelineEventCurator` matching
`/api/admin/orders/{uuid}/messages(?:/.*)?$`. Checked early in `curate()` immediately
after the items-segment skip. Returns `Optional.empty()` — same rationale as items HTTP skip.

### Clock-skew tolerance window
2-second window (`SKEW_TOLERANCE_SECONDS = 2`). Handler iterates
`messages.findAllByOrderIdOrderByCreatedAtAsc(orderId)` and takes the latest candidate
whose `created_at.atOffset(UTC)` is not after `audit.createdAt + 2s`. Covers DB DEFAULT
vs JPA Instant skew in practice.

### V002 seed user dependency removed
Initial test used `ownerActorId()` querying `misza@drshoes.pl`. In the full suite,
`AdminWebTestBase.cleanupUsers()` wipes all users including V002 seeds between tests.
Fixed by passing `actorId=null` to `sendManual` — acceptable, matches `MessageRouterIntegrationTest` pattern.

### trigger_id FK constraint
Test 2 originally used `UUID.randomUUID()` for triggerId — violates `message_trigger_id_fkey`.
Fixed by adding `seededTriggerId()` helper that queries `SELECT id FROM trigger_ WHERE enabled = TRUE LIMIT 1`
(V006 seeds 4 triggers; always present in the reused Testcontainer).
