## Decisions / deviations from plan

- **TestEndpoints inner class requires explicit @TestConfiguration wiring:**
  The plan states "@SpringBootTest scans the inner TestEndpoints controller automatically
  because it's a static nested class in a @SpringBootTest-annotated test." In practice
  this is not accurate for Spring Boot 3.x: @SpringBootTest uses the main application
  component-scan path, which does not include test-scoped classes. Without registration,
  /test/owner-only returned 404. Fix: added a static `TestEndpointsConfig` inner class
  annotated @TestConfiguration that declares a @Bean for TestEndpoints, and added
  @Import(TestEndpointsConfig.class) to RbacIntegrationTest. This is the standard
  Spring Boot pattern for test-only controllers. One additional import line vs. plan.

- **AfterEach cleanup applied:** Per lesson from 0b-6, added `@AfterEach void cleanup()
  { users.deleteAll(); }` in RbacIntegrationTest alongside the plan-provided BeforeEach
  seed. This prevents DB pollution of UserRepositoryIntegrationTest and other shared-DB
  tests. Required — not optional.

- **CSRF on /test/owner-only:** GET request — CSRF does not apply. Verified: the request
  passes through SecurityConfig's `.anyRequest().permitAll()` (endpoint not under
  /api/admin/**), the authenticated session cookie satisfies the session-based auth
  context, and @PreAuthorize fires on the controller method. No CSRF interaction at all.

- **Logging in RbacService at DEBUG, not INFO:** Per dispatch mandate §7. Performance-
  sensitive path (called on every secured invocation). Format:
  `op=rbacCheck capability={} actor={} role={} outcome={}`.

## Logging / metadata wired

- `RbacService` logs at **DEBUG** (not INFO — per task mandate, called per-request):
  - `op=rbacCheck capability={capability} actor=anonymous role=anonymous outcome=false` (for null/unauthenticated)
  - `op=rbacCheck capability={capability} actor={name} role={role} outcome={true|false}` (for authenticated)
- Password, session id, and credentials are never logged (irrelevant to this service but uniform policy maintained).

## Concerns

1. **Plan comment about auto-scan of inner @RestController was inaccurate.** Spring Boot 3.x
   @SpringBootTest does NOT auto-scan static inner classes of test files. The @TestConfiguration
   + @Import workaround is correct and clean, but it diverges from the plan's claim. The plan's
   note "if you find it isn't picked up, add a @TestConfiguration import" was the right fallback
   and was applied. No functional concern; documentation concern only.

2. **`prePostEnabled = true` is the default in Spring Security 6.x.** `@EnableMethodSecurity`
   enables pre/post security by default; the `prePostEnabled = true` attribute is redundant but
   harmless and matches the plan exactly. No action needed; flagged for awareness.

3. **Hardening debt (pre-existing, carried from 0b-4):** `server.servlet.session.cookie.secure=false`
   in `application.yaml`. Not introduced by this task. Still pending before milestone-0b tag.
