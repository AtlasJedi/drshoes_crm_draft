## Decisions / deviations from plan

**Prop interface extended beyond plan spec:**
Plan body shows `Props = { photo: Photo | null; onClose: () => void }` only.
Dispatch requirements also specified prev/next navigation and inline relabel.
Added three optional props: `photos?: Photo[]`, `onNavigate?: (p: Photo) => void`,
`onRelabel?: (p: Photo, label: PhotoLabel) => void`.
Props are optional so plan's minimal call-site stays valid; `OrderDrawerPhotos`
was updated to pass all four props.

**Radix Dialog is installed — used as planned:**
`@radix-ui/react-dialog@^1.1.15` confirmed in `apps/web/package.json`.
No substitution needed.

**Navigation model — parent owns state:**
`onNavigate(photo)` delegates navigation back to `OrderDrawerPhotos` (which holds
the `setOpen` state). This is the cleanest pattern: lightbox stays stateless re:
which photo is open; parent list owns the selection. Keyboard ArrowLeft/Right
call `onNavigate` when handler is wired.

**Inline relabel:**
Label shown as an underline button in the metadata strip. Click opens a `<select>`
(autoFocus). `onChange` fires `onRelabel(photo, newLabel)` and collapses back to
display mode. `onBlur` also collapses (e.g. Escape pressed inside the select).
Relies on `OrderDrawerPhotos.onRelabel` which already calls `relabelPhoto` + `refresh`.

**LOC: 116 — over 80 soft cap:**
Cohesion is tight (one component, one concern). No clean split point. Acknowledged.
Precedent: 3-10 PhotoUploader was 116 LOC, same allowance granted.

## Concerns
- Keyboard Escape handling: both the Dialog.Root `onOpenChange` and the `keydown`
  listener fire on Escape. Dialog.Root's `onOpenChange(false)` calls `onClose` first;
  keydown listener also calls `onClose`. Double-fire is benign (setState with same
  value) but slightly redundant. Could remove the keydown Escape branch; kept for
  explicit clarity in the log.
- Arrow key navigation button clicks only log + call `onNavigate`. The Dialog briefly
  unmounts and remounts as `photo` changes from parent. This causes a flash on slow
  connections. A future improvement would hold the lightbox open and only swap the
  `photo` prop without remounting, but that requires the parent to keep `open=true`
  while changing the photo — out of scope for this task.
