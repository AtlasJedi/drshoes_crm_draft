## Concerns
- `InboundMessageService` is exactly at the 120 non-blank cap. No margin for additions. If 5-5 or 5-6 need to extend this class, split by extracting to a separate helper class.
- `thread.setUnreadCount(thread.getUnreadCount() + 1)` is not race-safe under concurrent inbound webhooks (known M5 limitation, non-blocking per plan errata note 8).
- `findByPhone` on `ClientRepository` is an exact-match finder — phone must be stored in normalized E.164 form in the DB for this to work. If existing client records have un-normalized phone values, matching will silently fail. Carry forward to 5-5/5-6 authors.
