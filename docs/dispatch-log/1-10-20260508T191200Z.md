## Decisions / deviations from plan

### ORDER_CREATED synthesis
The POST /api/admin/orders audit row has path=/api/admin/orders (no orderId in path),
so the pathPrefix scan (/api/admin/orders/{orderId}%) does NOT match it.
Decision: synthesise ORDER_CREATED from Order.createdAt in AuditTimelineService.
Actor: resolved from the earliest actor found in the scanned audit rows (same owner).
If Order is not found (unknown orderId), no synthetic event is prepended — returns empty list.
M2 debt: direct lookup of the POST /api/admin/orders audit row to get the true actor
at create-time, or add a dedicated orderId field to the audit table.

### Repository query shape
`findOrderTimelineRows(String pathPrefix, UUID orderId)` takes pathPrefix as a String
parameter (e.g. "/api/admin/orders/{orderId}%") to keep the JPQL simple. The LIKE
clause uses Postgres LIKE semantics — % wildcard. The OR clause captures INTERNAL rows
via parentEntityId.

### RBAC
No @PreAuthorize annotation on the controller method — the existing SecurityConfig
admin filter chain already requires isAuthenticated() for all /api/admin/** paths.
Anonymous → 401 is proven by test 3. Both OWNER and EMPLOYEE have access.

### Controller package
Placed at com.drshoes.app.audit.api.AuditTimelineController per errata block.
This ensures AuditLogAspect pointcut fires for future write operations on this controller.

### actorFullName for actor test
AdminWebTestBase seeds fullName="Owner Test". The service batch-loads userRepo.findAllById()
and maps UUID → fullName. The STATUS_CHANGED row has actorId = owner's UUID, so
actorFullName = "Owner Test". Test asserts non-empty, confirmed passing.

### Service size

AuditTimelineController: 56 LOC (trivially small).

## Concerns

  including in the timeline). Acceptable M1 brittleness.
