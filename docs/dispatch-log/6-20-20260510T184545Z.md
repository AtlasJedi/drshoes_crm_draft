## Decisions

1. **Promise.allSettled instead of Promise.all:** mirrors the calendar page
   pattern (6-16). Board fetch failure shows ErrorBanner; triggers fetch failure
   is non-fatal — board renders with empty triggers array (no crash, dnd still
   works, trigger dialog will show no previews).

2. **KanbanCardDto version defaults to 0:** `KanbanCardDto` has no `version`
   field (by design in 6-8 spec). The `versionMap` defaults all cards to version
   0 via `(card as CardWithOptionalVersion).version ?? 0`. On a 409 conflict the
   hook's existing error handler surfaces an error toast. No backend slice needed
   for this task — the plan's version note was documented and the cast pattern
   from the plan spec was preserved exactly.

3. **ErrorBanner onRetry omitted in server component:** The existing
   `ErrorBanner` (shipped 6-12) uses `onRetry?: () => void` (client callback),
   not `retryHref`. Since `KanbanPage` is a Server Component, no callback can
   be passed. Using the component without `onRetry` is valid (the prop is
   optional); the user can reload the page to retry.

4. **Relative mock path corrected in test:** The plan's mock path was
   `"../../../_components/kanban/KanbanBoardWrapper"` (3 levels up = `admin/`),
   which is incorrect. The correct relative path from
   `kanban/__tests__/page.test.tsx` is `"../../_components/kanban/KanbanBoardWrapper"`
   (2 levels up = `orders/`). Fixed in test file.

5. **State primitives reused, not recreated:** `Skeleton`, `EmptyState`, and
   `ErrorBanner` were all present in `apps/web/components/state/` (shipped 6-12).
   The Skeleton API uses `height="h-N"` prop (not `className` for height), so
   individual `<Skeleton height="h-10" />` calls are used in the Suspense fallback.
