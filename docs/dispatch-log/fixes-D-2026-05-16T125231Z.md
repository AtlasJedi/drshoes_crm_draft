## Change
TimelineEventCurator.curate (the POST .../notes branch) previously gated on `status == 200`.
OrderNotesController.add returns 201 Created via `ResponseEntity.created(...)`, so every real
note row was silently classified as "unknown HTTP" and never emitted as an `ORDER_NOTE` timeline
event. As a result, notes never appeared in the order drawer's "Notatki wewnętrzne" section.

Predicate now accepts both 200 and 201.
