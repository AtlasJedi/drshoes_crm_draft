## Decisions / deviations from plan

1. **No `@radix-ui/react-dropdown-menu`** ‚Äî plan spec used Radix DropdownMenu for PhotoCard, but
   only `@radix-ui/react-dialog` is installed and the constraint says "no new dependencies."
   Replaced with a `useState`-driven inline menu (button + conditional div). Visually equivalent;
   no external dep needed.

2. **PhotoUploader and PhotoLightbox created as stubs** ‚Äî `OrderDrawerPhotos` imports both, so
   TypeScript would fail without them. Stubs have the correct prop signatures (match 3-10/3-11
   plan specs) and are marked with `TODO(3-10)` / `TODO(3-11)`. Tasks 3-10 and 3-11 will
   overwrite these with full implementations.

3. **Zdjƒôcia wired as section, not a tab strip** ‚Äî the existing OrderDrawer uses a flat vertical
   stack of sections (not Radix Tabs). The plan said "add Zdjƒôcia tab" and showed a tab-button
   pattern, but the existing drawer has no tab UI at all. Section was inserted between Timeline
   and Wiadomo≈õci using the same `<section className="px-6 py-4 border-t border-admin-line">`
   pattern as the other drawer sections. This mirrors the actual codebase conventions rather than
   introducing a tab shell that doesn't exist.

4. **`void refresh()` in `useEffect`** ‚Äî added `void` prefix to suppress the `no-floating-promises`
   lint rule (consistent with existing components like OrderDrawerTimeline).
