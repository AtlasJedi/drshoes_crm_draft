## Decisions / deviations from plan
- Plan ERRATA correctly identifies V004 as only `ALTER TABLE order_ ADD COLUMN version INTEGER NOT NULL DEFAULT 0` — done exactly as specified.
- `OrderSource` enum added (not in plan body) because V001 declares `source VARCHAR(20) NOT NULL DEFAULT 'ADMIN'`. Without a Java-side default the JPA INSERT would send NULL and fail the DB NOT NULL constraint. Entity defaults `source = OrderSource.ADMIN`.
- `@AfterEach cleanup()` added to test: deletes order_items → orders → client in FK-safe order. Required because the shared Testcontainers Postgres (`withReuse(true)`) bleeds state, and `AdminWebTestBase.seedUsers()` calls `clients.deleteAll()` which would fail if orders still reference those clients.
- Optimistic-locking smoke test (two EntityManager instances, second save throws `OptimisticLockingFailureException`) was **deferred to task 1-6** (OrderService). At the pure repository layer with `@SpringBootTest` and a single EntityManager context, manufacturing a genuine version conflict requires a second detached session which adds significant boilerplate for marginal test-layer coverage. The `@Version` annotation and V004 column are wired correctly; the conflict path is best exercised at the service integration test level.
