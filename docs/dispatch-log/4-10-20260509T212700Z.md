## Decisions

1. **Plan test bug fixed**: `retry_retryOfRetry_chainContinues` — plan's test helper used `failedMessage(retry1Id, ORDER_ID, 2, ...)` but the `attempt` param was unused, leaving retryAttempt at default 1. Without explicit `retry1Msg.setRetryAttempt(2)`, the service would compute nextAttempt=2, not 3. Added explicit `setRetryAttempt(2)` call after building retry1Msg.

2. **No new repository finder needed**: `MessageRepository.findById` (inherited from JpaRepository) is sufficient. MessageRouter.sendManual returns the new message's UUID; the service does a second findById to load it for linkage.

3. **@Audited(parent="#result.orderId()")**: SpEL uses `#result` (the returned MessageDto record); `orderId()` is the record accessor. AuditedParentResolver binds `#result` after method return — confirmed working per AuditedParentResolver.java lines 61-74.

4. **LOC**: MessageRetryService = 109 lines (under 120 cap; plan estimated ~60 but that excluded Javadoc and the toDto helper).

5. **No new migration**: V010 already added retry_of_message_id + retry_attempt to message table.

6. **MessageDto extended**: errorCode + errorMessage were missing from the record even though MessageEntity already had them. Added all four new fields. MessagesController.toDto() updated accordingly.
